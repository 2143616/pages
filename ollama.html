🧠 Ollama 是什么？

Ollama 是一个开源的本地大语言模型（LLM）运行框架，旨在让任何人都能以最简洁的方式在本地机器上下载、运行和管理开源大模型。它将复杂的模型权重下载、推理引擎配置、显存管理、多模型切换等底层细节封装为统一的 CLI 和 REST API，让开发者可以像使用 SaaS API 一样消费本地模型。

🚀 一键运行

ollama run llama3 — 一条命令即可下载并启动交互式对话。

📦 模型市场

内置模型库，支持 Llama、Gemma、Mistral、Qwen、DeepSeek 等数百个主流模型。

⚡ GPU 加速

自动检测 NVIDIA CUDA / AMD ROCm / Apple Metal，发挥硬件最佳性能。

🔌 API 兼容

提供原生 REST API + OpenAI & Anthropic 兼容 API，无缝替换云端服务。

🛠 自定义模型

通过 Modelfile 调整系统提示词、参数、模板，或导入 GGUF / Safetensors 模型。

🌐 跨平台

macOS（原生 + Metal）、Windows（x64）、Linux（x64 + ARM64）、Docker 全覆盖。

💡 核心理念： Ollama 不是模型，而是一个 模型运行引擎。你可以把它理解为"本地的 Docker + Hub，但对象是大模型"。它负责下载、缓存、推理、服务暴露，你只需要选模型、写提示词。

📥 安装指南

macOS

推荐方式——下载官方安装包：

# 方式一：官网下载 .dmg（推荐）
访问 https://ollama.com 点击 Download

# 方式二：Homebrew
brew install ollama

# 安装后启动服务（以后台守护进程运行）
ollama serve

Linux

# 一键安装脚本（支持 x86_64 & ARM64）
curl -fsSL https://ollama.com/install.sh | sh

# 验证安装
ollama --version
🐧 Linux 手动安装： 如遇到网络问题，可以下载 ollama-linux-amd64.tgz 或 ollama-linux-arm64.tgz 解压到 /usr/local。

Windows

访问 ollama.com 下载 OllamaSetup.exe，运行安装程序。安装完成后 Ollama 会自动在系统托盘中运行，并提供命令行入口。

🪟 Windows 注意： Ollama on Windows 需要 WSL2。安装程序会自动配置。如果已有 NVIDIA GPU，确保已安装 CUDA 驱动。

Docker

# CPU 版
docker run -d -v ollama:/root/.ollama -p 11434:11434 --name ollama ollama/ollama

# NVIDIA GPU 版（需要 nvidia-container-toolkit）
docker run -d --gpus all -v ollama:/root/.ollama -p 11434:11434 --name ollama ollama/ollama

# AMD GPU 版
docker run -d --device /dev/kfd --device /dev/dri \
  -v ollama:/root/.ollama -p 11434:11434 --name ollama ollama/ollama:rocm

# 进入容器运行模型
docker exec -it ollama ollama run gemma4
⚠️ 数据持久化： 必须挂载 -v ollama:/root/.ollama，否则容器销毁后下载的模型会丢失。模型默认下载到 ~/.ollama/models/。

常用参数

# 指定生成参数
ollama run gemma4 --temperature 0.7 --top-p 0.9 --seed 42

# 设置上下文长度（默认 2048，扩展至 8192）
ollama run qwen2.5:7b --num-ctx 8192

# 保持模型常驻内存（避免每次重新加载）
ollama run gemma4 --keep-alive 5m
💡 环境变量： 设置 OLLAMA_HOST=0.0.0.0:11434 可让服务监听所有网卡，
OLLAMA_MODELS=/path/to/models 自定义模型存储路径，
OLLAMA_NUM_PARALLEL=4 同时处理多个请求，
OLLAMA_KEEP_ALIVE=5m 模型闲置 5 分钟后卸载。

📜 Modelfile 自定义模型

Modelfile 是 Ollama 的模型构建配置文件，类似于 Dockerfile。你可以基于现有模型微调行为、嵌入系统提示词、调整推理参数，甚至导入全新模型。

基本结构

# 基于已有模型
FROM llama3.1:8b

# 设置系统提示词（角色设定）
SYSTEM """
你是一个专业的 Python 编程助手。回答要简洁、准确，包含代码示例。
"""

# 调整推理参数
PARAMETER temperature 0.3
PARAMETER top_p 0.9
PARAMETER num_ctx 4096

# 设置模板（可选，通常无需修改）
TEMPLATE """
{{ if .System }}<|start_header_id|>system<|end_header_id|>

{{ .System }}<|eot_id|>{{ end }}
<|start_header_id|>user<|end_header_id|>

{{ .Prompt }}<|eot_id|>
<|start_header_id|>assistant<|end_header_id|>

{{ .Response }}<|eot_id|>
"""

支持的参数

参数默认值说明
temperature0.8采样温度，越高越随机（范围 0~2）
top_p0.9核采样阈值，控制候选词累加概率
top_k40从概率最高的 K 个 token 中采样
num_ctx2048上下文窗口大小（token 数）
repeat_penalty1.1重复惩罚系数
seed随机随机种子，设置后结果可复现
stop—停止词，遇此词停止生成
num_predict128最大生成 token 数（-1 为不限）

构建并运行自定义模型

# 创建 Modelfile, 然后:
ollama create my-python-helper -f ./Modelfile

# 运行自定义模型
ollama run my-python-helper "写一个斐波那契数列生成器"

OpenAI 兼容 API

Ollama 同时暴露兼容 OpenAI 格式的端点。这意味着你可以在任何支持 OpenAI API 的工具中，只改 Base URL 即可使用本地模型：

# OpenAI 兼容端点
curl http://localhost:11434/v1/chat/completions -d '{
  "model": "gemma4",
  "messages": [{"role": "user", "content": "你好！"}],
  "stream": false
}'

# 嵌入兼容端点
curl http://localhost:11434/v1/embeddings -d '{
  "model": "nomic-embed-text",
  "input": "Hello world"
}'
🔑 使用场景： 将 VS Code 的 Cline / Continue、ChatGPT Next Web、OpenAI Python SDK、LangChain、Jupyter AI 等工具的 API Base URL 改为 http://localhost:11434/v1，即可零改动接入本地模型。

📚 官方库与集成

Python 库

# 安装
pip install ollama

# 基础流式对话
import ollama

stream = ollama.chat(
    model='gemma4',
    messages=[{'role': 'user', 'content': '讲个笑话'}],
    stream=True,
)
for chunk in stream:
    print(chunk['message']['content'], end='', flush=True)

# 嵌入向量
response = ollama.embed(model='nomic-embed-text', input='你好世界')
print(response['embeddings'][0][:5])  # 前 5 维

JavaScript / TypeScript 库

// 安装
npm install ollama

// 流式对话
import ollama from 'ollama'

const response = await ollama.chat({
  model: 'gemma4',
  messages: [{ role: 'user', content: '写一首诗' }],
  stream: true,
})
for await (const part of response) {
  process.stdout.write(part.message.content)
}

✨ 核心能力详解

🔄 流式输出 (Streaming)

Ollama 默认使用 Server-Sent Events（SSE）流式返回结果，用户无需等待完整生成，逐 token 实时显示。在 Python 中将 stream=True 即可接收流式迭代器。

curl http://localhost:11434/api/chat -d '{
  "model": "gemma4",
  "messages": [{"role":"user","content":"写一篇 500 字短文"}],
  "stream": true
}'
# 返回 SSE 格式，每行一个 token

🧠 思考模型 (Thinking / Reasoning)

支持 DeepSeek-R1 等"思维链"类模型。Ollama 会自动解析模型的 <think>...</think> 标记，将推理过程与最终答案分开展示：

ollama run deepseek-r1:7b "Solve: 1+1=?"

<think>
这是一个简单算术问题。1+1 在十进制中等于 2。不需要复杂推理。
</think>
答案是 2。

📐 结构化输出 (Structured Outputs)

通过 format 参数指定 JSON Schema，确保模型输出符合特定结构：

curl http://localhost:11434/api/chat -d '{
  "model": "gemma4",
  "messages": [{"role":"user","content":"北京天气"}],
  "format": {
    "type": "object",
    "properties": {
      "city":    {"type": "string"},
      "temp":    {"type": "number"},
      "weather": {"type": "string"}
    },
    "required": ["city", "temp"]
  }
}'
# 输出: {"city":"北京","temp":22,"weather":"晴"}
💡 适用场景： 函数调用参数解析、数据提取、表单生成等需要严格格式的场景。

👁️ 视觉理解 (Vision)

多模态模型（如 Llava、Gemma4-Vision、Qwen2-VL）可以直接输入图片进行分析：

# CLI 方式
ollama run llava "这张图片里有什么？" --image /path/to/photo.jpg

# API 方式
curl http://localhost:11434/api/chat -d '{
  "model": "llava",
  "messages": [{
    "role": "user",
    "content": "描述这张图片",
    "images": ["<base64_encoded_image>"]
  }]
}'

🔧 工具调用 (Tool Calling / Function Calling)

支持让模型调用外部函数，实现 Agent 工作流。模型会输出工具调用请求，由应用调度执行：

curl http://localhost:11434/api/chat -d '{
  "model": "gemma4",
  "messages": [{"role":"user","content":"纽约现在几点？"}],
  "tools": [{
    "type": "function",
    "function": {
      "name": "get_current_time",
      "description": "获取指定城市当前时间",
      "parameters": {
        "type": "object",
        "properties": {
          "city": {"type": "string"}
        },
        "required": ["city"]
      }
    }
  }]
}'

🌐 联网搜索 (Web Search)

Ollama 支持通过内置的联网搜索工具，让模型在回答前检索实时信息。需要先配置搜索 API key：

ollama run gemma4:cloud "今天的比特币价格是多少？"
# Cloud 模型会自动联网搜索获取实时信息

📊 嵌入向量 (Embeddings)

将文本转换为语义向量，用于 RAG（检索增强生成）和语义搜索：

curl http://localhost:11434/api/embed -d '{
  "model": "nomic-embed-text",
  "input": ["文档1", "文档2", "文档3"]
}'
# 推荐嵌入模型：nomic-embed-text, bge-m3, mxbai-embed-large

☁️ Cloud 云模式

Ollama 不仅限于本地。对于需要更大参数规模（如 70B、405B）的场景，Ollama 提供 Cloud 模式，将请求转发到云端 GPU：

# 本地启动交互式 chat，但使用云端更大的模型
ollama run gemma4:cloud

# API 调用云端模型
curl https://ollama.com/api/chat -d '{
  "model": "gemma4:cloud",
  "messages": [{"role":"user","content":"复杂推理问题"}]
}'

🚀 高级技巧

🔁 并发推理

通过环境变量控制并行度，多请求共享模型显存：

export OLLAMA_NUM_PARALLEL=4    # 最多同时处理 4 个请求
export OLLAMA_MAX_LOADED_MODELS=2 # 最多同时加载 2 个不同模型
ollama serve

📡 远程访问

# 监听所有网络接口（注意安全！）
export OLLAMA_HOST=0.0.0.0:11434
ollama serve

# 内网其他机器即可访问
curl http://192.168.1.100:11434/api/chat -d '{...}'
🔒 安全提醒： Ollama 默认没有认证机制。0.0.0.0 暴露到公网非常危险。建议用 Nginx 反代 + Basic Auth，或 Tailscale / WireGuard 建 VPN 后在内网使用。

❓ 常见问题 (FAQ)

Q: 模型下载速度慢/下载失败怎么办？

国内用户建议设置镜像源。Ollama 默认从 GitHub Releases 和 Docker Hub 下载。可以配置代理： 

# 设置 HTTP 代理
export HTTP_PROXY=http://127.0.0.1:7890
export HTTPS_PROXY=http://127.0.0.1:7890
ollama run gemma4

Q: 模型加载后占用多少显存？

以 4-bit 量化（Q4_K_M）为例：7B ~ 4.5GB，13B ~ 8GB，34B ~ 20GB，70B ~ 42GB。上下文窗口越大，额外显存越多（约 1MB/token）。使用 ollama ps 查看实时显存占用。

Q: 如何在 CPU 上运行？

Ollama 自动检测 GPU，如果没有检测到就自动回退到 CPU。CPU 推理速度显著慢于 GPU（7B 模型约 3-8 token/s），建议使用更小的量化版本如 qwen2.5:0.5b 或 phi3:mini。

Q: 报错 "llama.cpp: out of memory" 怎么办？

显存不足。解决方案：
① 使用更小量化的版本（如 Q3_K_M 甚至 Q2_K）
② 减小 num_ctx（设为 1024）
③ 使用更小的模型（如 7B → 3B）
④ 增加 OLLAMA_GPU_OVERHEAD 为更大的值
