一、背景：为什么需要视频 LLM？

1.1 从"看懂图片"到"看懂视频"的鸿沟

2023 年，多模态大模型在图像理解上取得了巨大突破——LLaVA 能用自然语言描述图片、回答关于图片的问题、甚至根据图片讲故事。但视频比图片复杂得多：

维度图片视频
信息载体单个静态画面连续帧序列（时间维度）+ 音频轨道
关键挑战空间理解（物体在哪、是什么）时空理解（谁做了什么、先后顺序、因果关系）
典型问题"这张照片里有什么？""视频中的人物在第三秒做了什么？为什么后来改变了方向？"
数据量1 张图 ≈ 几十 KB ~ 几 MB10 秒视频 ≈ 250 帧 × 每帧 ≈ 几十 MB，还需处理音频
核心矛盾：LLM 的上下文窗口有限（早期 LLaMA 仅 2048 token），但一段 10 秒的视频如果每帧都转成 token，轻松超出 100K token。如何用有限的 token 预算"压缩"视频信息，同时保留足够的时间细节——这是所有视频 LLM 必须解决的核心问题。

1.2 Video-LLaMA 诞生的背景

Video-LLaMA 由 DAMO Academy（阿里巴巴达摩院）和 NTU 联合提出，论文《Video-LLaMA: An Instruction-tuned Audio-Visual Language Model for Video Understanding》于 2023 年 6 月发布。它是首批同时处理视频画面和音频的开源多模态 LLM 之一。

二、Video-LLaMA 整体架构

2.1 一句话架构概览

Video-LLaMA = 视频编码器 + 音频编码器 + 两个 Q-Former + 一个冻结的 LLM

↘

→ 文本回答

两个独立的分支：视觉分支处理视频帧序列，音频分支处理伴音轨道。两者各自通过一个 Q-Former 将模态信息"翻译"成 LLM 能理解的 token，最后拼接在一起送入 LLaMA 生成回答。

三、视觉分支：从视频帧到视觉 Token

3.1 第一步：帧采样（Frame Sampling）

10 秒的视频有 300 帧（30fps），全部处理计算量太大，且相邻帧几乎一样。Video-LLaMA 的采样策略：

  • 均匀采样 8 帧：将视频等分为 8 段，每段取 1 帧
  • 每帧缩放到 224×224 分辨率
  • 8 帧可以覆盖视频的起承转合，同时 token 量可控
为什么是 8 帧？ 这是一个工程 trade-off。越多帧 → 越细腻的时间信息，但 token 数线性增长。8 帧是实验验证的"甜点"——能区分动作变化（如"拿起来→放下"），又不会超出 LLM 的上下文窗口。后续 Video-LLaMA 2 扩展到 32 帧。

3.2 第二步：ViT 编码

每帧（224×224 RGB 图像）通过一个冻结的 ViT-G/14（Vision Transformer, Giant, patch_size=14）编码：

  • 将图像切成 16×16 = 256 个 patch（每个 14×14 像素）
  • 加上 1 个 [CLS] token → 共 257 个 token
  • 输出：每帧 257 个 1024 维的视觉特征向量
⚠ 问题：8 帧 × 257 token/帧 = 2056 个视觉 token。直接塞进 LLM 会吃掉一半的上下文窗口。必须压缩。

3.3 第三步：Q-Former 压缩（最精妙的部分）

Q-Former 的核心思想来自 BLIP-2 论文。它用一个可训练的轻量级 Transformer 来"提问"冻结的视觉编码器，将冗余的 2056 个视觉 token 压缩为 32 个高信息密度的 query token。

工作原理类比：

  • ViT 输出 = 一本详细说明书（2056 页），内容极其冗余
  • Q-Former = 一个聪明的"读者"，带着 32 个问题去读这本书
  • 32 个 query token = 32 个问题的答案摘要，保留了关键信息，丢了冗余

Q-Former 内部结构：

Q-Former = Cross-Attention Transformer

输入:
  - 32 个可学习的 Query Token [32, 768]
  - ViT 视觉特征作为 Key/Value [2056, 1024]
  
结构:
  Query → Self-Attention（32 个 Q 互相关注）
       → Cross-Attention（Q 去 attend 视觉特征）
       → FFN
       → 输出 32 个精炼后的 token

输出: [32, 768]  ← 从 2056 压缩到 32，压缩比 64:1

Q-Former 的训练目标（第一阶段）：

  • 图文匹配（ITM）：判断 query token 和视觉特征是否描述同一内容
  • 图文对比学习（ITC）：拉近匹配的图文对，推远不匹配的
  • 文本生成（LM）：根据视觉特征生成描述文字

这三个 loss 共同训练，确保 query token 既包含视觉信息，又能被 LLM 理解。

四、音频分支：从音频波形到听觉 Token

4.1 为什么需要音频？

无声的视频丢失了大量信息：说话声、环境音、背景音乐、事件音效（如关门声、爆炸声）。Video-LLaMA 是首批意识到"理解视频需要同时理解声音"的模型。

4.2 ImageBind：六模态对齐的"万能编码器"

Video-LLaMA 使用 Meta 提出的 ImageBind（Girdhar et al., 2023）作为音频编码器。ImageBind 的独特之处——它把六种模态（图像、文本、音频、深度、热力、IMU）对齐到了同一个向量空间。

为什么选 ImageBind 而不是专门的音频模型（如 Whisper）？
ImageBind 的音频 embedding 和图像 embedding 在同一个空间——这意味着一个音频 token（"狗叫声"）和一张图片 token（"狗的照片"）的向量接近。这种跨模态对齐让 LLM 更容易将声音和画面关联起来（"视频里有狗叫 → 画面里应该有一条狗"）。

处理流程：

音频波形（16kHz, ~10s）
  ↓ 转为 128 通道的梅尔频谱图（Mel-Spectrogram）
  ↓ ImageBind Audio Encoder
  ↓ 输出音频特征序列
  ↓ Audio Q-Former（与 Vision Q-Former 结构相同）
  ↓ 8 个音频 query token [8, 768]

五、Q-Former：跨模态对齐的桥梁

5.1 为什么不用简单的 MLP 投影？

LLaVA 的做法是用一个 MLP 把视觉 token 直接映射到 LLM 的 embedding 空间。这对于图片足够——1 张图 257 token，MLP 映射后直接喂给 LLM。

但 Video-LLaMA 面临两个挑战：① 视频有 8 帧 ×256 = 2048 token，MLP 不压缩，token 数不变；② 视频帧之间有时间关系，MLP 无法建模"第 2 帧和第 7 帧之间的因果联系"。

Q-Former 一举两得：压缩 token 数 + 通过 Self-Attention 建模帧间关系。

5.2 Q-Former 内部 Forward 流程（逐步）

# 伪代码：Vision Q-Former 的一次前向传播

# 输入:
#   query_tokens: [32, 768]       — 32 个可学习的查询向量
#   video_features: [2056, 1024]  — ViT 输出的视觉特征

# Step 1: 投影维度对齐
video_features = Linear(1024 → 768)(video_features)   # [2056, 768]

# Step 2: Self-Attention — 32 个 query token 互相交流
# "第 3 个 query 可能关注全景，第 7 个关注运动"
query_tokens = SelfAttention(Q=query_tokens, K=query_tokens, V=query_tokens)

# Step 3: Cross-Attention — query 去"读取"视觉特征
# 每个 query 选择性地关注它感兴趣的视觉区域/帧
query_tokens = CrossAttention(Q=query_tokens, K=video_features, V=video_features)

# Step 4: FFN 非线性变换
query_tokens = FFN(query_tokens)

# 输出: 32 个 query token，每个都融合了全局视觉信息
层作用输入输出
维度投影将 ViT 的 1024 维对齐到 768 维（LLaMA embedding 维度）[2056, 1024][2056, 768]
Self-Attention让 32 个 query 分工协作（"你关注前景，我关注背景"）[32, 768][32, 768]
Cross-Attentionquery 从视觉特征中抽取信息Q:[32,768] KV:[2056,768][32, 768]
FFN增加非线性表达能力[32, 768][32, 768]

六、两阶段训练策略

6.1 阶段 1：多模态对齐预训练（Vision-Language Alignment）

项目详情
目的让 Q-Former 学会从视觉/音频特征中提取 LLM"能懂"的信息
训练数据大规模图文对（WebVid-2M：250 万视频-文本对 + CC3M 图像-文本对）
冻结ViT、ImageBind、LLaMA 全部冻结
训练仅 Q-Former + 维度投影层
LossITM + ITC + LM 三合一
类比教一个翻译官（Q-Former）把"视觉语言"翻译成"LLM 语言"，翻译官的参考词典就是图文对
💡 为什么用 250 万视频+CC3M 而不是纯视频数据？ 视频数据标注质量差（"一个人在海边散步"这种描述太模糊），静态图像的标注通常更精准。图文数据帮 Q-Former 建立扎实的视觉-语言映射基础，视频数据负责加上时间维度。

6.2 阶段 2：指令微调（Instruction Tuning）

项目详情
目的让模型学会"问答格式"——根据视频回答各种类型的问题
训练数据高质量视频问答数据集（VideoChat 指令数据、MiniGPT-4 图像指令数据混合）
冻结ViT、ImageBind 继续冻结
训练Q-Former + LLaMA 的 LoRA 适配器
Loss标准自回归 LM Loss（Next Token Prediction）
类比翻译官已经会翻译了，现在教它"别人问问题你要怎么回答"——看很多 QA 范例来学格式
🔥 关键设计：LLM 用 LoRA 微调而非全参训练
全参微调 7B 模型需要巨量显存和数据，且容易过拟合小数据集导致灾难性遗忘。LoRA 只在 Attention 的 Q/K/V 权重旁加低秩分解矩阵（参数量仅原来的 ~0.1%），既保留了预训练知识，又学到了"看视频回答问题"的能力。

九、能力与局限

✅ 擅长什么

  • 视频内容描述：能概括视频中发生了什么
  • 视听关联：能将画面和声音联系起来（"这个声音对应画面中的哪个动作"）
  • 时序问答："视频开头发生了什么？"
  • 多轮对话：能连续追问关于同一视频的问题

❌ 不擅长什么

  • 精细时序定位："第 3.5 秒到第 4.2 秒之间发生了什么"——8 帧采样不够
  • 长视频理解：>2 分钟的视频信息丢失严重
  • 复杂空间推理："画面中有几个红色物体？它们的位置关系？"
  • 多人物追踪：人物换装/遮挡后再出现时容易混淆

十、面试高频考点

10.1 必会 5 问

Q1: Video-LLaMA 用了几个 Q-Former？为什么？

两个独立的 Q-Former——一个处理视觉、一个处理音频。视觉和音频是不同模态，特征分布差异大，各自有专门的 query token 效果更好。共享 Q-Former 会导致两种特征互相干扰。

Q2: Q-Former 和 MLP 投影的核心区别？什么时候该用哪个？

MLP 只做维度映射，不改变 token 数。Q-Former 通过 Cross-Attention 压缩 token 数，适合 token 数多的场景。规则：图片（<500 token）→ MLP 够用；视频（>2000 token）→ 必须 Q-Former 压缩。

Q3: 为什么要冻结 LLM 只用 LoRA？

① 全参微调 7B 需要 >56GB 显存，LoRA 可在单卡完成；② 视频 QA 数据量小（数万条），全参容易过拟合；③ 冻结 LLM 保留预训练知识，LoRA 只学"看视频"这一增量能力。

Q4: 为什么要分两阶段训练？

阶段 1（对齐）：数据量大但质量低（250 万视频-文本对），用来让 Q-Former 学会基础的视觉→语言映射。阶段 2（指令微调）：数据量小但质量高（数万条精细 QA），用来学会问答格式。先粗后精，避免高质量数据被低质量数据淹没。

Q5: Video-LLaMA 2 相比第一代改进了什么？

① 帧数从 8 → 最多 32 帧；② 用更强的 ViT（SigLIP）替换 ViT-G；③ 引入 STC（Spatial-Temporal Convolution）模块更好地融合时空信息；④ 支持更长的视频（数分钟级别）；⑤ 训练数据规模大幅增加。
