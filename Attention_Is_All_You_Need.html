一、动机：为什么需要 Transformer？

1.1 RNN 的困境

在 2017 年之前，序列转导（sequence transduction）任务——机器翻译、文本摘要、语音识别——几乎被 RNN / LSTM / GRU 统治。RNN 的核心计算方式是：

ht = f(ht−1, xt)

这意味着每个时间步 t 的计算必须等前一步 t−1 完成。这带来两个致命问题：

  • 无法并行训练：训练时，一个长度为 N 的句子需要 N 步串行计算，GPU 的大量核心闲着。
  • 长距离依赖衰减：信息在 RNN 链条中传递时逐渐丢失，虽然 LSTM/GRU 有所缓解，但根本瓶颈仍在。

1.2 已有改进及其局限

此前已经有工作尝试用 CNN 替代 RNN（如 ByteNet、ConvS2S），利用卷积的并行优势。但它们引入了一个新问题：两个任意位置之间建立联系所需的操作次数随着距离增长（ConvS2S 是线性，ByteNet 是对数），这不利于长距离依赖的学习。

核心洞察：既然 Attention 机制已经能在任意两个位置之间建立直接联系（O(1) 路径长度），为什么不用 Attention 来彻底替代 RNN/CNN，构建一个纯粹基于注意力机制的模型？

二、模型架构总览

Transformer 延续了经典的 Encoder-Decoder 结构，但内部完全由 Self-Attention 和 Feed-Forward Network (FFN) 堆叠而成，没有任何 RNN 或卷积层。

三、注意力机制 —— 论文的数学核心

3.1 什么是 Attention？

从信息检索的视角理解 Attention：

  • Query（查询）：当前我想知道什么？（类比搜索引擎的搜索框）
  • Key（键）：每个位置能提供什么？（类比每篇文章的标题）
  • Value（值）：每个位置的实际内容是什么？（类比文章的正文）

Attention 的本质是：用 Q 和每个 K 计算相似度得到权重，再对 V 加权求和。

3.2 为什么叫"注意力"机制？

这个名字来自对人类认知的精确类比。

人类的注意力

你走进一间嘈杂的咖啡馆找朋友：

  1. 选择性聚焦：自动忽略墙上花纹、背景音乐、路过服务员……只关心"哪些人脸特征匹配我朋友"
  2. 分配权重：戴眼镜的男性 → 多看一眼；金发女性 → 扫过跳过
  3. 加权整合：最终你"看到"的是一个加权后的画面——朋友的脸最清晰，其他都是模糊背景

映射到数学

人类注意力→数学对应
"我朋友长什么样"（搜索意图）→Query (Q)
每张人脸的特征（被检索的索引）→Key (K)
每张人脸的实际视觉信息（被检索的内容）→Value (V)
大脑对各面孔的重视程度→softmax 权重
最终"看到"的加权画面→加权求和结果

一个翻译例子

翻译 "The cat sat on the mat" → "猫坐在垫子上"。

当模型生成"坐"这个字时，注意力分布（softmax 权重）可能是：

The    cat    sat    on    the    mat
0.01   0.02   0.85   0.03   0.01   0.08
                        ↑
                模型在"注意" sat

模型不是机械地把每个词逐一翻译，而是动态决定每时每刻应该"看"输入的哪些部分——就像人类译者会来回扫视原文一样。

论文中的可视化证明

论文附录图 3–5 展示了注意力头实际学到的行为：

  • 某个头始终追踪动词 making 和远方宾语 more difficult 的依赖关系
  • 某个头精准锁定代词 its 指向 The Law

这就是"注意力"——模型真的在选择性地关注句子中相关的部分，而不是无差别地处理所有词。所以这个名字不是比喻，而是对机制工作方式的精确描述：它让模型学会该"注意"哪里。

3.3 Scaled Dot-Product Attention

Attention(Q, K, V) = softmax( Q KT ) V
                             √dk

计算过程（以单个 token 为例）：

  1. 点积算相似度：把当前 token 的 Q 向量与序列中所有 token 的 K 向量做点积 → 得一个分数向量。
  2. 缩放：除以 √dk。为什么？当 dk 很大时，点积数值会很大，softmax 后会落入梯度极小的"饱和区"。除以 √dk 使方差控制为 1。
  3. Softmax：将分数转为概率分布（所有权重之和为 1，每个在 0~1 之间）。
  4. 加权求和：用这些权重对 V 加权求和，得到最终输出。
💡 为什么是点积而非加性 Attention？ 点积可以利用高度优化的矩阵乘法，在 GPU 上极快；加性 Attention 需要额外的前馈网络计算，更慢且更占内存。

3.4 Multi-Head Attention（多头注意力）—— 关键创新

与其只做一次 Attention（dmodel=512 维），不如将 Q/K/V 分别投影到 h=8 个低维子空间（每个 64 维），每个子空间独立做 Attention，最后拼接起来再投影一次。

MultiHead(Q, K, V) = Concat(head1, ..., headh) · WO
其中   headi = Attention(Q·WiQ, K·WiK, V·WiV)

为什么要多头？一个直观类比：

🔍 单头 Attention

像用一盏聚光灯看句子，只能捕捉一种关系模式（比如"主语-谓语"）。

🔍🔍 多头 Attention

像用 8 盏不同颜色的聚光灯同时照亮句子，不同头可以关注不同模式——有的头关注语法结构，有的关注指代关系，有的关注语义相似度。

论文附录的可视化（图 3-5）漂亮地证实了这一点：某些头专门处理长距离依赖（如动词"making"和远方宾语"more difficult"的联系），某些头则精确捕捉代词指代（"its"指"the Law"）。

计算量不增加：每个头维度为 dk=dmodel/h=64，总计算量与单头全维度 Attention 几乎相同。

3.5 Attention 如何用到模型中？—— 完整数据流

以翻译 "I love you" → "我爱你" 为例，走一遍 Attention 在模型中的实际运作。

整体流程

输入: "I love you" │ ┌────▼──────────────────────────────────┐ │ ENCODER（6 层） │ │ 每层: Self-Attention → FFN │ │ 作用: 理解输入，每个词融入上下文信息 │ └──────────────────┬───────────────────┘ │ 编码后的上下文表示 ┌──────────────────▼───────────────────┐ │ DECODER（6 层） │ │ 每层: │ │ Masked Self-Attention → │ │ Encoder-Decoder Attention → │ │ FFN │ │ 作用: 逐词生成输出 │ └──────────────────┬───────────────────┘ │ 输出: "我 爱 你"

1️⃣ Encoder Self-Attention —— "理解输入"

输入 "I", "love", "you"，每个词都要通过 Attention 和其他词交互：

"I" 的 Q 去问所有词的 K：
      I    love  you
   ├──────┼──────┼──────┤
   0.1   0.6   0.3   ← 权重（"I" 最关心 "love"）

"I" 的输出 = 0.1×VI + 0.6×Vlove + 0.3×Vyou
           → 此时的 "I" 已经混入了 "love" 和 "you" 的信息

每个词都做这件事。6 层堆叠后，"I" 的表示包含了整句话的上下文。

2️⃣ Decoder Masked Self-Attention —— "不能偷看未来"

解码是自回归的——一次生成一个词，当前词只能看到之前已生成的词：

时刻 1: 输入 <start>         → 生成 "我"
时刻 2: 输入 <start> 我       → 生成 "爱"
时刻 3: 输入 <start> 我 爱   → 生成 "你"

Mask 的作用：把未来位置的权重设为 −∞，softmax 后变成 0：

计算 "爱" 的 Attention 时：
     <start>   我    爱    你
     0.3    0.7    0    0   ← "你" 被 mask 掉，看不到
     ✅     ✅     ✅    ❌

3️⃣ Encoder-Decoder Attention —— "对齐输入和输出"

这是连接 Encoder 和 Decoder 的关键桥梁——Decoder 词从 Encoder 的输出中提取相关信息：

Decoder 在生成 "爱" 时：
   Q 来自 Decoder（"我现在要生成'爱'了"）
   K, V 来自 Encoder 最终输出（"I love you" 的上下文表示）

注意力权重:   I    love   you
              0.05  0.9  0.05
                     ↑
            最关注 "love"——翻译正确！

🔢 注意力权重到底是怎么算出来的？

注意力权重的核心公式：

Attention(Q, K, V) = softmax( Q KT ) V
                             √dk

下面用这个翻译例子，逐步拆解 0.9 这个权重是怎么来的。

第 1 步：从向量开始

假设 dk = 4，Encoder 已编码完 "I love you"，每个词对应一个向量：

词Encoder 输出向量（同时也是 K 和 V）
Ie₁ = [0.5, 0.2, 0.8, 0.1]
lovee₂ = [0.3, 0.9, 0.1, 0.7]
youe₃ = [0.6, 0.1, 0.4, 0.9]

Decoder 当前正在生成 "爱"，它的隐藏状态经线性变换后作为 Q：

q = [0.2, 0.8, 0.3, 0.5]
💡 直觉：Q 编码的是 Decoder 当前的需求——"我现在需要一个词，它和主语有关"。
K 编码的是 Encoder 输出中每个词的"标签"——"我是 love，我有这个含义"。

第 2 步：计算 Q 与每个 K 的点积（相似度分数）

scorei = Q · Ki
score("I") = 0.5×0.2 + 0.2×0.8 + 0.8×0.3 + 0.1×0.5 = 0.10 + 0.16 + 0.24 + 0.05 = 0.55 score("love") = 0.3×0.2 + 0.9×0.8 + 0.1×0.3 + 0.7×0.5 = 0.06 + 0.72 + 0.03 + 0.35 = 1.16 score("you") = 0.6×0.2 + 0.1×0.8 + 0.4×0.3 + 0.9×0.5 = 0.12 + 0.08 + 0.12 + 0.45 = 0.77

"love" 得分最高 (1.16)，因为 Q 和 love 的 K 向量在第 2、4 维上都很大——"方向一致"。

第 3 步：缩放（除以 √dk）

scaled_scorei = scorei / √dk = scorei / 2
scaled("I")     = 0.55 / 2 = 0.275
scaled("love") = 1.16 / 2 = 0.580
scaled("you")   = 0.77 / 2 = 0.385
🤔 为什么要缩放？当 dk 很大时（如 512），点积数值会变得很大，softmax 后梯度趋近于 0（梯度消失）。除以 √dk 把方差控制为 1，让训练稳定。

第 4 步：Softmax → 得到注意力权重 α

αi = escaled_scorei / Σj escaled_scorej
e0.275 ≈ 1.317 e0.580 ≈ 1.787 e0.385 ≈ 1.470 总和 = 1.317 + 1.787 + 1.470 = 4.574
词计算注意力权重 α
I1.317 / 4.5740.29
love1.787 / 4.5740.39
you1.470 / 4.5740.32
📝 注：这个简化例子中（dk=4，随机模拟）权重差距不大。在真实的 512 维 Transformer 里，经过充分训练后，语义匹配的那个词会得到压倒性的权重——就像上面例子中的 0.9。那不是魔法，而是反向传播不断调整 WQ、WK 投影矩阵，让模型学会：当 Decoder 要生成 "爱" 时，它的 Q 向量天然就和 "love" 的 K 向量高度对齐。

第 5 步：用权重加权求和 V → 最终输出

output = 0.29 · VI + 0.39 · Vlove + 0.32 · Vyou

这个输出向量被送入 Decoder 的下一层，最终投影到词表上——它携带了来自 "love" 最多（但不排他）的信息。

四、位置编码 —— 让无序的 Attention 感知顺序

Attention 机制本身是置换不变的——把句子打乱，Attention 输出不变。但语言依赖于词序。Transformer 通过在输入词向量上直接相加位置编码来注入位置信息。

4.1 正弦位置编码

PE(pos, 2i)   = sin( pos / 100002i / dmodel )
PE(pos, 2i+1) = cos( pos / 100002i / dmodel )

其中 pos 是词在句子中的位置（0, 1, 2, ...），i 是维度索引（0, 1, ..., dmodel/2-1）。

精妙之处：

  • 不同频率的正弦波组成不同维度——低维变化慢（编码全局位置），高维变化快（编码局部位置）。
  • 对于任意固定偏移 k，PEpos+k 可以表示为 PEpos 的线性函数，这使得模型可以轻松学习相对位置关系。
  • 使用固定函数而非可学习参数：效果几乎相同（Table 3 行 E），但正弦编码可以外推到训练时未见过的更长序列。
🧠 直觉理解：想象一个只有两个维度的简化位置编码——位置 0 得 (0,1)，位置 1 得 (0.84,0.54)，位置 2 得 (0.91,-0.42)... 这些向量在单位圆上均匀旋转。每个位置有一个独一无二的"角度指纹"，而且邻近位置的角度也邻近。

五、其他关键组件

5.1 Position-wise Feed-Forward Network (FFN)

FFN(x) = max(0, x·W1 + b1)·W2 + b2

每个位置的 FFN 参数相同，但层与层之间参数不同。可以理解为两层的 1×1 卷积。它给模型增加了非线性变换能力。维度变化：512 → 2048 → 512。

5.2 残差连接 + Layer Normalization

output = LayerNorm(x + Sublayer(x))

每个子层（Attention 或 FFN）的输出都会先加上输入（残差连接），再做 LayerNorm。这极大地缓解了深层网络的梯度消失问题。

5.3 权重共享

Encoder 输入词向量、Decoder 输出词向量、最后 Linear 层的权重矩阵三者共享。类似 Inception 网络的做法，减少参数、提升效果。

六、为什么选择 Self-Attention？

论文从三个维度比较了 Self-Attention、RNN 和 CNN：

层类型每层计算复杂度最少串行操作数最长依赖路径
Self-AttentionO(n²·d)O(1) ✅O(1) ✅
RNN (LSTM/GRU)O(n·d²)O(n) ❌O(n) ❌
CNN (ConvS2S)O(k·n·d²)O(1)O(logk n)
  • 并行度：Self-Attention 的 O(1) 串行操作意味着所有位置可同时计算，充分利用 GPU。
  • 长距离依赖：O(1) 的最长路径意味着任意两个位置之间信息只需一步就能传递。RNN 需要 O(n) 步。
  • 复杂度：当序列长度 n 小于表示维度 d（在 NLP 中通常成立），Self-Attention 比 RNN 更快。只有在超长序列场景（如图像像素级）O(n²) 才成为瓶颈，论文建议未来用局部受限 Attention 解决。

8.3 英文句法分析

Transformer 在 WSJ Penn Treebank 句法分析上达到 92.7 F1（半监督），且即使仅用 40K 训练句也能超越专门的 BerkeleyParser。这证明 Transformer 不仅在翻译任务上优秀，也有很好的泛化能力。

九、为什么这篇论文改变了世界？

🔑 范式转变

证明了"Attention only"不仅可行，而且更好。RNN/CNN 不再是必选项。

⚡ 可并行性

训练时间从数周缩短到数小时，使研究者能更快迭代、尝试更大规模。

📈 可扩展性

"越大越好"的特性为大模型时代埋下伏笔——后来的 GPT、BERT 直接受益于这一架构。

🧩 通用性

从 NLP 扩展到 CV（ViT）、语音、多模态——Transformer 成为跨领域的通用计算范式。

演化树

Transformer (2017) ├── BERT (2018) — 仅用 Encoder，双向理解，NLU 之王 ├── GPT (2018 → GPT-4, 2023) — 仅用 Decoder，单向生成，语言模型之王 ├── T5 (2019) — Encoder-Decoder，统一 NLU+NLG ├── ViT (2020) — 将图像切成 Patch 当作 Token，Attention 进入视觉 └── LLaMA / Claude / Gemini — 几乎所有现代大语言模型都是 Transformer

局限与后续方向

  • O(n²) 复杂度：序列长度 n 增大时计算量爆炸，推动了 Longformer、BigBird、FlashAttention 等高效变体。
  • 缺少归纳偏置：相比 CNN 的平移不变性，Transformer 没有预设的归纳偏置，需要更多数据训练。
  • 位置编码：正弦编码之后出现了 RoPE（旋转位置编码）、ALiBi 等更优方案。

十、一句话总结

用"全注意力"替代"全循环"——这一简单的想法构建出了更并行、更强大、更通用的 Transformer 架构，彻底改写了现代深度学习的版图。
