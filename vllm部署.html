🚀 Step-3.7-Flash 部署指南

在 DGX Spark 上通过 vLLM + 4-bit 量化部署 143B MoE 多模态大模型
vLLM 4-bit 量化 DGX Spark 128GB

1. 整体方案

我们最终选择

组件选择原因
模型 Step-3.7-Flash ~143B 总参，~7B 激活，比 3.5 更小更新，支持多模态
推理引擎 vLLM 原生 CUDA 加速，PagedAttention 省显存，社区最活跃
量化方式 bitsandbytes 4-bit 动态量化，无需提前转 GGUF，加载即用
硬件 DGX Spark 128GB 统一内存，4-bit 下约 72GB 装得下

内存估算

部分大小说明
模型权重 (4-bit)~72 GB143B × 0.5 字节
KV Cache (8K context)~8 GBMoE 8K 上下文估计
中间激活 / 其他~10 GBMoE 路由、注意力计算等
总计~90 GBDGX Spark 128GB ✅ 充裕

2. 环境准备

Step 1: 确认 CUDA 环境

DGX Spark 预装了 CUDA。验证一下：

# 检查 CUDA 版本
nvidia-smi
# 期望看到 CUDA Version: 12.x 或更高

# 检查 GPU
nvcc --version
💡 如果没有 nvidia-smi，先装 NVIDIA 驱动：sudo apt install nvidia-driver-545（具体版本看 DGX Spark 官方文档）。

Step 2: 安装 Python 环境与 vLLM

推荐用 conda 创建独立环境：

# 创建 Python 3.11 环境
conda create -n step37 python=3.11 -y
conda activate step37

# 安装 PyTorch（确认 CUDA 版本）
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124

# 验证 PyTorch 和 GPU
python -c "import torch; print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0))"
# 应输出：True 和 Grace Blackwell 或类似

安装 vLLM 和 bitsandbytes：

# 安装 vLLM（推荐从源码编译以支持 Grace 架构）
pip install vllm

# 如果上面报错，试试预编译 wheel
# pip install https://github.com/vllm-project/vllm/releases/...

# 安装 bitsandbytes（4-bit 量化依赖）
pip install bitsandbytes

# 验证安装
python -c "import vllm; print(vllm.__version__)"
python -c "import bitsandbytes; print(bitsandbytes.__version__)"
⚠️ 如果 pip install vllm 失败，可能是 Grace ARM 架构没有预编译包。备选方案：
1. 从源码编译：git clone https://github.com/vllm-project/vllm.git && cd vllm && pip install -e .
2. 或使用 SGLang 替代：pip install sglang[all]

安装 HuggingFace 相关工具：

# 用于下载模型
pip install huggingface-hub

# 登录（可选，公开模型不需要）
huggingface-cli login

3. 部署模型

Step 3: 下载模型权重

从 HuggingFace 下载 Step-3.7-Flash 的原始权重（Safetensors 格式）：

# 方法一：用 huggingface-cli 下载（推荐，支持断点续传）
huggingface-cli download stepfun-ai/Step-3.7-Flash \
  --local-dir ./models/Step-3.7-Flash \
  --local-dir-use-symlinks False

# ⚠️ 注意：下载约 286GB（BF16），确保磁盘有 >400GB 空间
💡 磁盘空间提醒：原始 BF16 权重 ~286GB。4-bit 加载只需要 ~72GB 内存，但下载时需要 286GB 磁盘空间。建议准备一个 512GB+ 的 SSD。

Step 4: 启动服务（方式一：vLLM 命令行）

vLLM 内置了 OpenAI 兼容的 API 服务器：

# 启动 OpenAI 兼容 API 服务
python -m vllm.entrypoints.openai.api_server \
  --model ./models/Step-3.7-Flash \             # 或用 HF ID: stepfun-ai/Step-3.7-Flash
  --trust-remote-code \                           # 必须！StepFun 自定义架构
  --quantization bitsandbytes \                   # 启用 4-bit 量化
  --load-format bitsandbytes \                    # bitsandbytes 加载格式
  --max-model-len 8192 \                          # 限制上下文长度，省内存
  --gpu-memory-utilization 0.9 \                  # 使用 90% 统一内存
  --dtype auto \                                  # 自动选择数据类型
  --port 8000                                     # API 端口
⚠️ 第一次启动会做什么？
1. 加载模型配置 → 2. 实例化自定义架构 → 3. 下载并加载权重 → 4. bitsandbytes 实时量化到 4-bit → 5. 启动 API 服务
整个过程取决于磁盘读取速度，约 5-15 分钟。

Step 4b: 启动服务（方式二：Python 脚本）

如果需要更多控制，写一个 Python 启动脚本：

# serve_step37.py
from vllm import LLM, SamplingParams
from vllm.entrypoints.openai.api_server import run_server
import uvicorn

# 配置模型
llm = LLM(
    model="stepfun-ai/Step-3.7-Flash",    # 或本地路径 ./models/Step-3.7-Flash
    trust_remote_code=True,                   # 自定义架构
    quantization="bitsandbytes",              # 4-bit 量化
    load_format="bitsandbytes",
    max_model_len=8192,                      # 上下文长度
    gpu_memory_utilization=0.9,               # 内存使用率
    dtype="auto",
)

# 启动 OpenAI 兼容 API 服务器
# 监听 http://0.0.0.0:8000
run_server(
    llm=llm,
    host="0.0.0.0",
    port=8000,
)
# 运行
python serve_step37.py
💡 启动后，服务会在 http://localhost:8000 提供 OpenAI 兼容 API，可以用任意 OpenAI SDK 调用。

5. 调优

内存调优

参数推荐值说明
--max-model-len 4096 ~ 16384 值越小，KV Cache 越省内存。简单问答 4096 就够
--gpu-memory-utilization 0.85 ~ 0.95 DGX Spark 统一内存，设到 0.9 安全
--max-num-batched-tokens 4096 ~ 8192 控制每批处理的 token 数
--enforce-eager 可选 关闭 CUDA graph，省内存但降低速度

批量与并发

# 优化吞吐量的启动参数
python -m vllm.entrypoints.openai.api_server \
  --model ./models/Step-3.7-Flash \
  --trust-remote-code \
  --quantization bitsandbytes \
  --load-format bitsandbytes \
  --max-model-len 8192 \
  --gpu-memory-utilization 0.9 \
  --max-num-batched-tokens 8192 \
  --max-num-seqs 8 \              # 最多同时处理 8 个请求
  --port 8000

6. 常见问题

Q：ImportError: cannot import name 'bitsandbytes'
A：安装 bitsandbytes：pip install bitsandbytes。如果 ARM 架构不支持，试试 pip install bitsandbytes --prefer-binary。
Q：RuntimeError: trust_remote_code=True is required
A：启动时加上 --trust-remote-code，因为 Step-3.7-Flash 使用了自定义模型架构。
Q：CUDA out of memory
A：降低 --max-model-len（如 4096）和 --gpu-memory-utilization（如 0.8）。如果还是 OOM，考虑用 Q3 量化或换更小的模型。
Q：模型回答速度很慢
A：这是 143B MoE 模型的正常现象，4-bit 量化本身会增加解码延迟。可以：
1. 减少 --max-model-len
2. 使用 --enforce-eager
3. 考虑加 --num-scheduler-steps 优化调度
Q：vLLM 不支持 ARM (Grace) 架构
A：DGX Spark 是 ARM64 架构。如果 vLLM 没有预编译包：
1. 从源码编译：git clone https://github.com/vllm-project/vllm.git && cd vllm && pip install -e .
2. 或使用 SGLang：pip install sglang[all]
3. 或先用 HuggingFace Transformers + bitsandbytes 跑（慢但能用）
备用方案：HuggingFace Transformers 直接加载 
# 如果 vLLM 搞不定，用 Transformers + bitsandbytes
from transformers import AutoModelForCausalLM, AutoProcessor, BitsAndBytesConfig
import torch

quant_config = BitsAndBytesConfig(load_in_4bit=True)

model = AutoModelForCausalLM.from_pretrained(
    "stepfun-ai/Step-3.7-Flash",
    trust_remote_code=True,
    quantization_config=quant_config,
    device_map="auto",
    torch_dtype=torch.bfloat16,
)

processor = AutoProcessor.from_pretrained(
    "stepfun-ai/Step-3.7-Flash",
    trust_remote_code=True,
)
