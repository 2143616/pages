1 项目概览

FPro（Frequency Prompt Guided Transformer）是南开大学发表于 ECCV 2024 的一项图像复原工作。其核心思想是：在 Transformer 编解码器的各个阶段注入"频率提示"（Frequency Prompt）， 使网络能够显式地感知并利用图像中的低频结构信息和高频细节信息。

🎯 解决的问题

  • 去雨（Derain）
  • 去雨滴（Raindrop Removal）
  • 去雾（Dehaze）
  • 去运动模糊（Deblur）
  • 去摩尔纹（Demoire）

⚡ 核心创新

  • Split Frequency Module — 动态通道滤波器分离高低频
  • Frequency-Specific Prompt Generator — 生成频率专属提示
  • 高低频提示融合模块 — Attention 机制注入主网络
  • 无需额外的频率分支网络，轻量高效

2 背景与动机

2.1 图像复原的挑战

图像复原任务（去雨、去雾、去模糊等）的目标是从退化观测 \(y\) 中恢复出干净的图像 \(x\)。 这是一个典型的逆问题（ill-posed inverse problem）， 需要强大的先验知识来约束解空间。传统方法依赖于手工设计的先验（如稀疏性、总变分）， 而深度学习方法则从大量数据中隐式学习这些先验。

2.2 频率域视角

图像退化在不同频率分量上的表现不同：

  • 低频（平滑区域、背景）：雾霾、模糊主要影响低频成分
  • 高频（边缘、纹理、细节）：雨线、噪声、摩尔纹主要影响高频成分

然而，现有的基于 Transformer 的图像复原方法（如 Restormer）在空间域统一处理所有特征， 没有显式区分频率信息。FPro 的核心洞察在于： "让网络知道自己正在处理什么频率的内容，并据此调整特征"。

💡 核心思路
在 Encoder-Decoder 的瓶颈层，通过动态通道滤波器将特征分离为低频部分和高频部分， 然后分别生成"频率提示"，通过 Cross-Attention 机制将这些提示注入到解码器的每一层， 引导网络关注适合该层的频率信息。

如上图所示，FPro 采用标准的3层编码器-解码器架构，并在瓶颈处插入 SplitFrequencyModule 对第一层编码器的浅层特征进行高低频分离， 然后通过 PromptModule 将频率提示注入解码器的每一层以及最终的细化模块。

📌 为什么用第一层编码器的特征做频率分离？
第一层编码器输出的特征保留了最多的空间细节（分辨率最大），且尚未被大量下采样， 从中提取的高低频信息最为丰富和准确。高层特征经过多次下采样已经丢失了精细的空间结构。

4 核心模块详解

4.1 Transformer Block（基础构件）

FPro 的每个 Transformer Block 包含两个核心子模块，继承自 Restormer：

① MDTA — 多头转置自注意力

class Attention(nn.Module):
    def __init__(self, dim, num_heads, bias):
        self.qkv = nn.Conv2d(dim, dim*3, 1, bias=bias)
        self.qkv_dwconv = nn.Conv2d(dim*3, dim*3, 3, padding=1, groups=dim*3, bias=bias)
        self.project_out = nn.Conv2d(dim, dim, 1, bias=bias)

    def forward(self, x):     # x: (B, C, H, W)
        qkv = self.qkv_dwconv(self.qkv(x))
        q, k, v = qkv.chunk(3, dim=1)
        # 转置注意力: 对 channels 维度做注意力
        q = rearrange(q, 'b (head c) h w -> b head c (h w)', head=num_heads)
        k = rearrange(k, 'b (head c) h w -> b head c (h w)', head=num_heads)
        v = rearrange(v, 'b (head c) h w -> b head c (h w)', head=num_heads)
        attn = (q @ k.transpose(-2,-1)) * temperature
        out = (attn.softmax(dim=-1) @ v)
        return self.project_out(rearrange(out, ...))
❓ 为什么用"转置注意力"而非标准注意力？
标准自注意力的计算复杂度是 \(O(H^2W^2)\)，对于高分辨率图像不可接受。 转置注意力将计算复杂度降为 \(O(C^2)\)（C 是通道数，通常远小于 H×W）， 使得 Transformer 能直接处理整张图像而无需分块。

② GDFN — 门控深度可分离前馈网络

class FeedForward(nn.Module):
    def __init__(self, dim, ffn_expansion_factor, bias):
        hidden = int(dim * ffn_expansion_factor)
        self.project_in  = nn.Conv2d(dim, hidden*2, 1, bias=bias)
        self.dwconv      = nn.Conv2d(hidden*2, hidden*2, 3, padding=1,
                                     groups=hidden*2, bias=bias)
        self.project_out = nn.Conv2d(hidden, dim, 1, bias=bias)

    def forward(self, x):
        x1, x2 = self.dwconv(self.project_in(x)).chunk(2, dim=1)
        x = F.gelu(x1) * x2          # 门控机制
        return self.project_out(x)

4.2 编码器-解码器

编码器由 3 个 Stage 组成，分辨率逐级降为 1/4（通过 PixelUnshuffle 下采样）。 解码器通过 PixelShuffle 上采样，并通过 skip-connection 拼接编码器对应层的特征。 每个 Stage 包含若干 TransformerBlock（编码器部分关闭了自注意力 isAtt=False， 只保留 GDFN；解码器打开自注意力）。

4.3 Downsample / Upsample

下采样使用 Conv2d(通道→通道/2) + PixelUnshuffle(2) 的组合， 将空间分辨率减半的同时通道数翻倍。上采样则是对称的 Conv2d(通道→通道×2) + PixelShuffle(2)。

5.1 SplitFrequencyModule — 动态通道滤波器

class dynamic_filter_channel(nn.Module):
    def __init__(self, inchannels, kernel_size=3, group=8):
        self.conv = nn.Conv2d(inchannels, group*k_size², 1, bias=False)
        # 从全局平均池化生成通道级滤波器权重

    def forward(self, x):
        low_filter = self.ap_1(x)              # B, C, 1, 1 — 全局统计
        low_filter = self.conv(low_filter)     # B, G*k², 1, 1
        low_filter = sigmoid(gate(low_filter)) # 门控调制
        low_filter = softmax(low_filter)       # 沿空间维度归一化

        x_unfold = unfold(x, k_size)           # B, G, C/G, k², HW
        low_part = Σ(x_unfold * low_filter)    # 加权求和 → 低频分量
        out_high = x - low_part                # 残差 → 高频分量
        return low_part, out_high
🔑 工作原理
这个模块本质上学习了一个数据驱动的低通滤波器：通过全局池化感知当前图像的内容， 动态生成 8 组（group=8）3×3 的卷积核权重，用 Softmax 归一化后对每个局部窗口加权求和。 结果 low_part 相当于对输入做了一次自适应低通滤波， 而 out_high = x - low_part 则提取出高频残差。

5.2 FrequencySpecificPromptGenerator (FSPG)

FSPG 对低频和高频采用不同的生成策略：

🔵 低频提示 FSPG_low

在频域中处理：

y = torch.fft.rfft2(feature)  # 实FFT
y_f = cat([real, imag])
y_att = DWConv + GELU(y_f)    # 自门控
y_f = y_f * y_att              # 特征调制
y = complex(real, imag)
y = y * learnable_weight       # 可学习权重
y = torch.fft.irfft2(y)       # 反变换回空间域

通过可学习的复数权重在频域中放大/抑制特定频率分量。

🟠 高频提示 FSPG_high

在空间域中处理：

ffm = interpolate(feature, size=(H,W))
y_att = DWConv_3×1 + GELU +
        DWConv_1×3(ffm)        # 分离式卷积
y_f = y_att * ffm              # 自门控调制
y = y_f * learnable_weight     # 可学习缩放

通过可学习的空间权重强调/弱化特定空间位置的高频细节。

两种提示生成器都有一个可学习的权重张量（self.weight / self.complex_weight），在训练过程中自动学习哪些频率/空间位置对复原任务更重要。

5.3 提示注入 — PromptModule

生成的提示通过两种 Attention 机制注入到主干特征中：

低频率提示融合（lowFrequencyPromptFusion）

# 全局 Cross-Attention
query = Q_proj(decoder_feature)          # 解码器特征作为 Query
key   = K_proj(avg_pool(prompt_low))     # 低频提示池化后作为 Key
value = V_proj(avg_pool(prompt_low))     # 全局上下文建模
out = softmax(query @ key.T / τ) @ value
# 输出: 被低频提示调制后的解码器特征

高频率提示融合（highFrequencyPromptFusion）

# 窗口级 Cross-Attention
query_feature_win = window_partition(decoder_feature, 8)  # 8×8 窗口
kv_feature_win    = window_partition(prompt_high, 8)
# 在每个窗口内做自注意力
query = Q_proj(query_feature_win)
key   = K_proj(kv_feature_win)
value = V_proj(kv_feature_win)
out = softmax(query @ key.T / scale) @ value
out = window_reverse(out)                # 恢复为完整特征图
# 输出: 被高频提示引导后的解码器特征
🧩 为什么低频用全局注意力，高频用窗口注意力？
低频信息是全局性的（整体亮度、颜色、背景结构），适合用全局池化和全局注意力捕获。 高频信息是局部性的（边缘、纹理、细节），在局部窗口内做注意力更高效， 且能更好地保留精细的空间结构。这种非对称设计是 FPro 的巧妙之处。

5.4 整体前向流程

def forward(self, inp_img):
    # 编码器
    enc1 = encoder1(patch_embed(inp_img))   # (B, 48, H, W)
    enc2 = encoder2(down1_2(enc1))          # (B, 96, H/2, W/2)
    enc3 = encoder3(down2_3(enc2))          # (B, 192, H/4, W/4)

    # 频率分离（从 enc1 提取）
    low_part, out_high = splitFre(enc1)

    # 解码器 + 频率提示
    dec3 = decoder3(enc3)
    dec3 = prompt_d3(low_part, out_high, dec3) + dec3    # 残差注入

    dec2 = decoder2(cat([up3_2(dec3), enc2]))
    dec2 = prompt_d2(low_part, out_high, dec2) + dec2

    dec1 = decoder1(cat([up2_1(dec2), enc1]))
    dec1 = prompt_d1(low_part, out_high, dec1) + dec1

    # 细化
    out = refinement(dec1)
    out = prompt_r(low_part, out_high, out) + out

    return output(out) + inp_img  # 全局残差连接

6.2 损失函数

# 像素级 L1 损失
l_pix = L1Loss(pred, gt)                    # weight=1.0

# 频域 FFT 损失 — 强制网络恢复正确的频率分布
pred_fft = torch.fft.fft2(pred, dim=(-2,-1))
target_fft = torch.fft.fft2(gt, dim=(-2,-1))
l_fft = L1Loss(pred_fft, target_fft)        # weight=0.1

# 总损失
l_total = l_pix + 0.1 * l_fft
💡 FFT 损失的作用
FFT 损失在频域中计算 L1 距离，迫使预测结果的幅度谱和相位谱与 Ground Truth 一致。 这能有效恢复图像的高频细节（边缘、纹理）并避免过度平滑。 权重 0.1 确保频域约束是辅助性的，主导仍然是空间域的 L1 损失。

6.3 混合增强 (Mixing Augmentation)

class Mixing_Augment:
    def mixup(self, target, input_):
        lam = Beta(1.2, 1.2).sample()       # 从 Beta 分布采样混合系数
        r_index = random_permutation(B)     # 随机选取另一张图
        target = lam * target + (1-lam) * target[r_index]
        input_ = lam * input_ + (1-lam) * input_[r_index]
        return target, input_

Mixup 以概率 50% 应用，能有效增加训练数据的多样性，提高模型的泛化能力。 去雾任务使用了 mixup（去雾性能提升明显），而去雨任务则禁用了 mixup。

7.1 大图像测试策略

测试时，FPro 使用重叠分块 + 加权合并的策略处理高分辨率图像： 

# 1. 将图像切分为 256×256 的块，块之间重叠 158/200 像素
split_data, starts = splitimage(input_, crop_size=256, overlap_size=158)

# 2. 对每个块独立推理
for i, data in enumerate(split_data):
    split_data[i] = model_restoration(data)

# 3. 加权合并：中心区域权重更高，边缘权重递减
restored = mergeimage(split_data, starts, crop_size=256, resolution=(B,C,H,W))
# 合并权重使用高斯得分图或均匀得分图

这种策略避免了直接输入大图时的显存溢出问题，同时通过重叠区域的加权平均消除了块边界伪影。

10.3 损失函数 — losses.py

class FFTLoss(nn.Module):
    # 在频域计算 L1 损失
    def forward(self, pred, target):
        pred_fft = torch.fft.fft2(pred, dim=(-2, -1))
        pred_fft = torch.stack([pred_fft.real, pred_fft.imag], dim=-1)
        target_fft = torch.fft.fft2(target, dim=(-2, -1))
        target_fft = torch.stack([target_fft.real, target_fft.imag], dim=-1)
        return l1_loss(pred_fft, target_fft)

FFT 损失将预测和真实图像的频域表示（实部和虚部分开）进行 L1 比较。 这强制网络同时恢复正确的幅度谱和相位谱。

12 实战：手机拍摄屏幕视频的去摩尔纹

这是很多人的真实痛点：用手机拍摄电脑/电视屏幕时，画面会出现彩色的摩尔纹（Moire Pattern）。 FPro 项目自带的 Demoire（去摩尔纹）模块恰好解决这个问题。 本章提供从环境搭建到视频处理的全流程手把手教程。

⚠️ 一个重要发现
查看 FPro 项目 Demoiring/Options/RealDemoiring_FPro.yml 官方配置文件发现：
去摩尔纹任务官方使用的是 type: Restormer（而非 FPro）。这意味着：
  • 可以直接用：Restormer 在 TIP18 数据集上表现已经很好
  • 也可以换成 FPro：FPro 的频域提示机制理论上对摩尔纹这种高频退化更有效，但需要自己训练
  • 下面的方案给出两条路径，你可以根据自己的需求选择

12.1 摩尔纹的形成原理

📐 摩尔纹 = 空间混叠
当手机相机的传感器像素网格与屏幕的子像素排列（RGB stripe / Pentile / Delta） 在空间频率上接近时，会发生差频干涉，产生肉眼可见的彩色波纹。 频率越接近，摩尔纹越明显；拍摄角度和距离的变化会导致摩尔纹形态不断变化 — 这也是视频比单图更难处理的原因。

📱 影响摩尔纹的因素

  • 手机与屏幕的夹角（越大越明显）
  • 手机分辨率 vs 屏幕分辨率
  • 屏幕刷新率 vs 手机帧率
  • 是否开启 HDR
  • 变焦倍数（数码变焦放大摩尔纹）

🎬 视频特有的挑战

  • 滚动快门（Rolling Shutter）产生时变伪影
  • 手持抖动导致摩尔纹形态不断变化
  • 逐帧处理可能出现闪烁
  • 需要处理大量帧（30fps × 60s = 1800 帧）

将以下代码保存为 Demoiring/process_video.py，然后运行： 

#!/usr/bin/env python3
"""
手机拍摄屏幕视频 → 去摩尔纹 → 输出视频
支持批量处理，自动分块，进度条显示

用法：
    python process_video.py \
        --input  video.mp4 \
        --output video_demoired.mp4 \
        --weights models/demoire.pth \
        --device cuda
"""

import cv2, torch, os, yaml, argparse, math
import numpy as np
from tqdm import tqdm

# ── FPro 架构（想用 Restormer 可以互换） ──
from basicsr.models.archs.FPro_arch import FPro
# from basicsr.models.archs.Restormer_arch import Restormer

parser = argparse.ArgumentParser()
parser.add_argument('--input',  required=True, help='输入视频路径')
parser.add_argument('--output', required=True, help='输出视频路径')
parser.add_argument('--weights', default='./models/demoire.pth')
parser.add_argument('--yaml', default='Options/RealDemoiring_FPro.yml')
parser.add_argument('--device', default='cuda')
parser.add_argument('--max_w', type=int, default=1920,
                    help='长边最大尺寸，超出的自动缩放（省钱省显存）')
args = parser.parse_args()

# ── 1. 加载模型 ──
print("[1/4] 加载模型...")
with open(args.yaml) as f:
    cfg = yaml.load(f, Loader=yaml.FullLoader)
cfg['network_g'].pop('type')

# 二选一：FPro 或 Restormer
model = FPro(**cfg['network_g'])
# model = Restormer(**cfg['network_g'])

ckpt = torch.load(args.weights, map_location=args.device)
# 兼容 'params' 和 'state_dict' 两种键名
sd = ckpt.get('params', ckpt.get('state_dict', ckpt))
model.load_state_dict(sd, strict=False)
model = model.to(args.device).eval()
print("   模型加载完成，参数数:", sum(p.numel() for p in model.parameters()))

# ── 2. 读取视频 ──
print("[2/4] 读取视频...")
cap = cv2.VideoCapture(args.input)
fps  = int(cap.get(cv2.CAP_PROP_FPS))
w    = int(cap.get(cv2.CAP_PROP_FRAME_WIDTH))
h    = int(cap.get(cv2.CAP_PROP_FRAME_HEIGHT))
total_frames = int(cap.get(cv2.CAP_PROP_FRAME_COUNT))

# 如果分辨率过大，等比缩放
scale = 1.0
if max(w, h) > args.max_w:
    scale = args.max_w / max(w, h)
    w, h = int(w * scale), int(h * scale)
    print(f"   原始分辨率 {cap.get(3):.0f}×{cap.get(4):.0f}"
          f"，缩放至 {w}×{h}")

fourcc = cv2.VideoWriter_fourcc(*'mp4v')
out = cv2.VideoWriter(args.output, fourcc, fps, (w, h))

# ── 3. 逐帧处理 ──
print(f"[3/4] 开始处理 {total_frames} 帧...")
for idx in tqdm(range(total_frames), desc='去摩尔纹'):
    ret, frame = cap.read()
    if not ret: break

    # 缩放
    if scale != 1.0:
        frame = cv2.resize(frame, (w, h), interpolation=cv2.INTER_AREA)

    # BGR → RGB → Tensor [0,1]
    img = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB).astype(np.float32) / 255.0
    t   = torch.from_numpy(img).permute(2, 0, 1).unsqueeze(0).to(args.device)

    # 填充至 8 的倍数（Restormer 要求）
    h_orig, w_orig = t.shape[2:]
    pad_h = (8 - h_orig % 8) % 8
    pad_w = (8 - w_orig % 8) % 8
    t = torch.nn.functional.pad(t, (0, pad_w, 0, pad_h), mode='reflect')

    # 推理
    with torch.no_grad():
        out_t = model(t)

    # 去除填充
    out_t = out_t[:, :, :h_orig, :w_orig]

    # Tensor → numpy → BGR
    out_t = out_t.clamp(0, 1).squeeze(0).permute(1, 2, 0).cpu().numpy()
    out_frame = cv2.cvtColor((out_t * 255).astype(np.uint8), cv2.COLOR_RGB2BGR)

    # 时域平滑（可选）：与上一帧以 0.1 的比例融合，减少闪烁
    if idx > 0:
        out_frame = cv2.addWeighted(out_frame, 0.9, prev_frame, 0.1, 0)
    prev_frame = out_frame.copy()

    out.write(out_frame)

cap.release()
out.release()
print(f"[4/4] ✅ 完成！已保存至 {args.output}")

官方 Demoiring 用的是 Restormer。但你完全可以换成 FPro 架构， 因为 FPro 的频域提示机制天然适合处理摩尔纹这种高频退化。 只需要在 YAML 文件中改一个字段：

  inp_channels: 3
  out_channels: 3

  ffn_expansion_factor: 3
  bias: False
  LayerNorm_type: WithBias
  dual_pixel_task: False

# 在 YAML 中指定预训练权重

⚡ 加速技巧
  1. FP16 推理：加上 model.half() 和 t.half()，速度翻倍，显存减半
  2. 降低分辨率：手机拍摄的 4K 视频可以先缩放到 1080p（`--max_w 1080`）， 摩尔纹主要在高频细节中，分辨率降低后摩尔纹反而减轻，处理速度大幅提升
  3. 跳帧处理：对于变化缓慢的屏幕内容，每隔 2-3 帧处理一帧，中间帧用插值
  4. TensorRT 导出：将模型导出为 TensorRT 引擎，1080p 可达 60+ FPS
  5. 批量推理：一次 batch 处理 4-8 帧，充分利用 GPU 并行能力

12.7 常见问题

❓ 问题：处理后的视频画面变模糊了
原因：去摩尔纹本质上是一个图像重建过程，过度去摩尔纹可能损失部分纹理细节。
解决：可以尝试在时域平滑中降低融合权重（0.9→0.7），或者在后处理中加一层 USM 锐化。
❓ 问题：有些帧的摩尔纹去掉了，有些还有残留
原因：手机抖动导致摩尔纹频率变化，模型对某些频率组合泛化不足。
解决：使用方案 C 微调。从你的视频中提取这些"难例"帧作为微调数据。
❓ 问题：显存不够（OOM）
解决：加 --max_w 720 进一步缩小分辨率； 或者在脚本中加上 model.half() 使用 FP16 推理； 或者在脚本中启用分块模式（参考 test_FPro.py 中的 splitimage + mergeimage）。
