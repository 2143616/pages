💡 1. Git 核心概念

Git 是目前世界上最先进的分布式版本控制系统，由 Linus Torvalds 在 2005 年为 Linux 内核开发而创作。

三个核心概念

📸 快照 (Snapshot)

Git 每次 commit 保存的是整个项目的快照，而非差异（diff）。这让分支切换和回退极快

🌳 分支 (Branch)

轻量级的指针，指向某个 commit。创建/切换分支几乎瞬时——仅改指针

🌐 分布式

每个开发者本地都有完整的仓库历史。断网也能 commit，在线才推送

三个区域

Git 工作区域
  ┌──────────┐    git add    ┌──────────┐    git commit   ┌──────────┐
  │  工作区   │─────────────→│  暂存区   │───────────────→│  仓库    │
  │ (Working  │              │ (Index / │                │ (History)│
  │  Directory)              │  Staging  │                │          │
  └──────────┘              └──────────┘                └──────────┘
        ↑                        ↑                          ↑
  你看到的文件            下次 commit 的内容          所有历史记录

# 核心命令
git init            # 创建仓库
git add .           # 工作区 → 暂存区
git commit -m "msg" # 暂存区 → 仓库
git checkout .      # 仓库/暂存区 → 工作区（覆盖）

def demo_basic_git():

🌿 4. 分支管理

分支是 Git 最强大的特性——轻量、快速、易合并。每个人都应该大胆创建分支。

4.1 分支操作

bash
# 查看分支
$ git branch                  # 本地分支列表（* 表示当前）
$ git branch -r               # 远程分支
$ git branch -a               # 所有分支

$ git branch -m old-name new-name

# 删除分支
$ git branch -d feature-login     # 已合并的
$ git branch -D feature-login     # 强制删除（未合并）

# 查看各分支最新提交
$ git branch -v
$ git log --oneline --graph --all --decorate

Python
def list_branches(path="."):
    """列出所有本地分支及当前分支"""
    r = subprocess.run(["git", "branch"], cwd=path,
                      capture_output=True, text=True)
    branches = r.stdout.strip().split("\n")
    current = [b for b in branches if b.startswith("*")][0].replace("*", "").strip()
    print(f"🔀 当前分支: {current}")
    print("所有分支:")
    for b in branches:
        marker = "→" if b.startswith("*") else "  "
        print(f"  {marker} {b.replace('* ', '').strip()}")
    return current, [b.replace("* ", "").strip() for b in branches]

🔀 5. 合并与冲突

5.1 Git Merge

bash
# 把 feature 合并到当前分支（通常是 main）
$ git checkout main
$ git merge feature
# → 无冲突：自动创建合并提交（Fast-forward 或 3-way）
# → 有冲突：手动解决后继续

# 禁止快进合并（保留分支历史）
$ git merge --no-ff feature -m "merge: 合并登录功能"

# 压缩合并（将 feature 的所有 commit 压缩成一个）
$ git merge --squash feature
$ git commit -m "feat: 登录功能"

# 中止合并（解决冲突到一半想放弃）
$ git merge --abort

5.2 冲突解决

冲突标记
# 当两个分支修改了同一个文件的同一区域时：
<<<<<<< HEAD
这是 main 分支的版本
=======
这是 feature 分支的版本
>>>>>>> feature

# 编辑文件 → 解决冲突 → 保存
$ git add .                   # 标记为已解决
$ git commit -m "解决冲突"      # 完成合并

# 可视化解决冲突工具
$ git mergetool               # 打开配置的合并工具（vscode, meld, ...）
Python
def simulate_conflict():
    """模拟一个合并冲突并用 Python 自动解决"""
    with tempfile.TemporaryDirectory() as tmpdir:
        os.chdir(tmpdir)
        subprocess.run(["git", "init"], capture_output=True)

        # 创建初始文件
        with open("config.py", "w") as f: f.write("TIMEOUT = 30\n")
        subprocess.run(["git", "add", "."])
        subprocess.run(["git", "commit", "-m", "初始提交"])

        # 创建 feature 分支并修改
        subprocess.run(["git", "checkout", "-b", "feature"])
        with open("config.py", "w") as f: f.write("TIMEOUT = 60\n")
        subprocess.run(["git", "commit", "-am", "feat: 增加超时"])

        # 切回 main 分支也修改同一文件
        subprocess.run(["git", "checkout", "main"])
        with open("config.py", "w") as f: f.write("TIMEOUT = 120\n")
        subprocess.run(["git", "commit", "-am", "fix: 修改超时"])

        # 合并（必然冲突）
        r = subprocess.run(["git", "merge", "feature"], capture_output=True, text=True)
        print("合并结果:", r.stdout)

        # 自动解决冲突：取两个版本的平均值
        with open("config.py") as f:
            content = f.read()

        import re
        vals = [int(x) for x in re.findall(r"TIMEOUT = (\d+)", content)]
        if vals:
            avg = sum(vals) // len(vals)
            with open("config.py", "w") as f:
                f.write(f"TIMEOUT = {avg}\n")
            subprocess.run(["git", "add", "config.py"])
            subprocess.run(["git", "commit", "-m", "解决冲突: TIMEOUT 取平均值"])
            print(f"✅ 冲突已解决，TIMEOUT = {avg}")

⏪ 7. 撤销与回退

Git 最让人心安的一点：几乎所有操作都可以撤销。

7.1 工作区/暂存区撤销

bash
# 撤销工作区的修改（还没 add）
$ git checkout -- file.py     # 丢弃工作区修改
$ git restore file.py         # Git 2.23+ 推荐
$ git checkout .              # 丢弃所有未暂存的修改

# 撤销暂存区的文件（已经 add，还没 commit）
$ git reset HEAD file.py      # 从暂存区移除
$ git restore --staged file.py # Git 2.23+ 推荐

# 撤销所有暂存
$ git reset HEAD .

7.2 提交撤销

bash
# --soft: 只移动 HEAD，文件留等待区
$ git reset --soft HEAD~1     # 撤销上一次 commit，修改留在暂存区

# --mixed (默认): 移动 HEAD，暂存区也被重置
$ git reset HEAD~1            # 撤销 commit 和暂存区，文件不变

# --hard: 彻底丢弃（⚠ 慎用，丢失工作区修改）
$ git reset --hard HEAD~1     # 完全回到上一个 commit

# 保留修改但撤销 commit（常用）
$ git reset --soft HEAD~1
# 然后可以重新提交

# 用 revert 创建"反提交"（更安全，不影响历史）
$ git revert HEAD             # 生成一个新的 commit 撤销当前修改
$ git revert <hash>           # 撤销特定 commit

7.3 reset vs revert 对比

git resetgit revert
效果删除历史创建反提交
历史被改写（push 后不要用）保留完整历史
适用本地未推送已推送的公共分支
安全⚠ 可能丢失修改✅ 安全

7.4 删除最近的 commit 但保留文件

bash
# 场景：刚 commit 了但发现有 bug，想改完重新提交
$ git reset --soft HEAD~1
# 修改文件
$ git add .
$ git commit -m "fixed: 修正提交"
Python
def undo_last_commit(soft=True):
    """撤销上一次 commit"""
    mode = "--soft" if soft else "--hard"
    r = subprocess.run(["git", "log", "--oneline", "-1"],
                      capture_output=True, text=True)
    print("撤销前最新的 commit:", r.stdout.strip())

    subprocess.run(["git", "reset", mode, "HEAD~1"])
    print(f"✅ 已撤销 (mode={mode})")

📦 8. Stash 临时保存

你正在功能分支上开发，突然需要切到 main 修复紧急 bug——但又不想 commit 半成品。

bash
# 保存当前工作区（包括未暂存和暂存文件）
$ git stash -u                # -u 包括未跟踪文件
$ git stash                   # 只保存已跟踪文件

# 查看 stash 列表
$ git stash list
# stash@{0}: On feature: 添加用户模型
# stash@{1}: On main: 临时保存

# 恢复最近一次 stash 并删除它
$ git stash pop

# 恢复 stash 但不删除
$ git stash apply stash@{1}

# 删除 stash
$ git stash drop stash@{1}    # 删除指定
$ git stash clear             # 清空所有

# 创建一个新的分支并应用 stash
$ git stash branch new-feature stash@{0}
Python
def stash_manager(action="list", message=""):
    """Stash 管理工具"""
    if action == "push":
        cmd = ["git", "stash", "push", "-u", "-m", message]
        subprocess.run(cmd)
        print(f"📦 已保存: {message}")

    elif action == "list":
        r = subprocess.run(["git", "stash", "list"], capture_output=True, text=True)
        if r.stdout.strip():
            print("📋 Stash 列表:")
            for line in r.stdout.strip().split("\n"):
                print("  ", line)
        else:
            print("📭 没有 stash")

    elif action == "pop":
        subprocess.run(["git", "stash", "pop"])
        print("↩️ 已恢复并删除 stash")

🔃 9. Rebase 变基

Rebase 是将分支的"基点"移动到另一个位置，获得干净、线性的历史。是 merge 的替代方案。

9.1 基本 Rebase

bash
# 场景：把 feature 分支变基到 main 的最新 commit
$ git checkout feature
$ git rebase main

# 效果：feature 的 commit 从 main 的末尾"重新播放"
# 之前:  A---B---C (main)
#             \
#              D---E (feature)
# 之后:  A---B---C (main)
#                  \
#                   D'---E' (feature)  ← 新的 hash

9.2 交互式 Rebase（修改历史）

bash
# 修改最近的 N 个 commit
$ git rebase -i HEAD~3

# 打开的编辑器中有如下选项：
pick abc123 提交1
pick def456 提交2
pick ghi789 提交3

# 可选指令：
# pick   — 保留该 commit
# reword — 修改提交信息
# edit   — 修改提交内容
# squash — 与上一个 commit 合并
# fixup  — 类似 squash 但丢弃提交信息
# drop   — 删除该 commit

9.3 常用 Rebase 场景

bash
# 1. 把最后 3 个 commit 压缩成 1 个
$ git rebase -i HEAD~3
# 把后两个改为 squash

# 2. 修改历史提交信息
$ git rebase -i HEAD~5
# 把要修改的 commit 改为 reword

# 3. 删除某个历史 commit
$ git rebase -i HEAD~10
# 把要删除的 commit 改为 drop

# 4. pull 时用 rebase 保持线性历史
$ git pull --rebase
# 配置为默认行为：git config --global pull.rebase true

9.4 Merge vs Rebase 对比

MergeRebase
历史保留合并点，可见分叉线性历史，无分叉
安全✅ 不修改已有 commit⚠ 修改 commit hash
适用公共分支合并私有分支整理
冲突解决一次完成每个 commit 都要解决一次
⚠️ 黄金法则：绝不要在已推送的公共分支上 rebase。rebase 会修改历史，别人推送到同一分支会混乱。

def squash_last_commits(n=3, message="合并提交"):
    """自动压缩最近 N 个 commit（需要 GIT_SEQUENCE_EDITOR 支持）"""
    import os
    # 设置编辑器为自动合并的脚本
    editor_cmd = f"sed -i '2,{n}s/^pick/squash/'"
    env = os.environ.copy()
    env["GIT_SEQUENCE_EDITOR"] = editor_cmd

    # 如果有冲突，需要手动处理
    print(f"✅ 已压缩最近 {n} 个 commit")

def create_version_tag(version, message=""):
    """创建版本标签并验证"""
    # 检查是否是有效的语义版本号
    import re
    if not re.match(r"^v?\d+\.\d+\.\d+", version):
        raise ValueError("版本号格式错误，应如 v1.0.0")

    # 创建带注释的标签
    msg = message or f"版本 {version}"
    subprocess.run(["git", "tag", "-a", version, "-m", msg])

    # 验证
    r = subprocess.run(["git", "tag", "-l", version], capture_output=True, text=True)
    print(f"🏷 标签 {r.stdout.strip()} 已创建")
    return version

def list_recent_tags(n=10):
    """按时间排序最近的标签"""
    r = subprocess.run(["git", "tag", "--sort=-creatordate"],
                      capture_output=True, text=True)
    tags = [t for t in r.stdout.strip().split("\n") if t]
    print(f"最近的 {min(n, len(tags))} 个标签:")
    for t in tags[:n]:
        r2 = subprocess.run(["git", "log", "-1", "--format=%ci", t],
                           capture_output=True, text=True)
        print(f"  {t:15} {r2.stdout.strip()}")

#!/usr/bin/env python3
"""pre-commit 钩子：检查 Python 语法和 .env 泄露"""
import sys, subprocess, os

# 1. Python 语法检查
result = subprocess.run(["python3", "-m", "py_compile", "main.py"],
                       capture_output=True, text=True)
if result.returncode != 0:
    print("❌ Python 语法错误:")
    print(result.stderr)
    sys.exit(1)

# 2. 检查是否误提交了 .env 文件
result = subprocess.run(["git", "diff", "--cached", "--name-only"],
                       capture_output=True, text=True)
staged_files = result.stdout.strip().split("\n")
for f in staged_files:
    if ".env" in f and f != ".env.example":
        print(f"❌ 可能泄露敏感信息: {f}")
        sys.exit(1)

print("✅ pre-commit 检查通过")

def auto_bisect(good_commit, bad_commit="HEAD", test_cmd="python -m pytest"):
    """自动二分查找（使用 git bisect run）"""
    # 创建测试脚本
    with open("bisect_test.sh", "w") as f:
        f.write(f"#!/bin/bash\n{test_cmd}\n")
    os.chmod("bisect_test.sh", 0o755)

    subprocess.run(["git", "bisect", "start"])
    subprocess.run(["git", "bisect", "bad", bad_commit])
    subprocess.run(["git", "bisect", "good", good_commit])
    r = subprocess.run(["git", "bisect", "run", "bash", "bisect_test.sh"],
                      capture_output=True, text=True)

    print(r.stdout[-500:])
    return r.stdout

import re
from datetime import datetime

def auto_changelog(since_tag=None):
    """自动生成 CHANGELOG（基于 conventional commits）"""
    if since_tag:
        range_arg = f"{since_tag}..HEAD"
    else:
        range_arg = "--all"

    r = subprocess.run(
        ["git", "log", "--oneline", range_arg],
        capture_output=True, text=True
    )

    commits = r.stdout.strip().split("\n")
    categories = {
        "feat": [], "fix": [], "docs": [],
        "refactor": [], "perf": [], "test": [], "other": []
    }

    for line in commits:
        match = re.match(r"([a-f0-9]+) (\w+)(\(.+\))?!?: (.+)", line)
        if match:
            _, type_, _, msg = match.groups()
            categories.get(type_, categories["other"]).append(msg)
        else:
            categories["other"].append(line)

    # 生成 Markdown
    labels = {
        "feat": "🚀 新功能",
        "fix": "🐛 Bug 修复",
        "docs": "📖 文档",
        "refactor": "♻️ 重构",
        "perf": "⚡ 性能",
        "test": "🧪 测试",
        "other": "📝 其他",
    }

    changelog = [f"# Changelog\n\n## {datetime.now().strftime('%Y-%m-%d')}\n"]
    for key, label in labels.items():
        if categories[key]:
            changelog.append(f"### {label}\n")
            for msg in categories[key]:
                changelog.append(f"  - {msg}")
            changelog.append("")

    content = "\n".join(changelog)
    with open("CHANGELOG.md", "w", encoding="utf-8") as f:
        f.write(content)
    print("✅ CHANGELOG.md 已生成")
    return content

def auto_release(version, branch="main"):
    """自动完成版本发布流程"""
    print(f"🚀 开始发布 {version}...")

    # 1. 确保在目标分支
    r = subprocess.run(["git", "rev-parse", "--abbrev-ref", "HEAD"],
                      capture_output=True, text=True)
    if r.stdout.strip() != branch:
        print(f"❌ 不在 {branch} 分支上")
        return

    # 2. 检查是否有未提交的修改
    r = subprocess.run(["git", "status", "--porcelain"], capture_output=True, text=True)
    if r.stdout.strip():
        print("❌ 有未提交的修改，请先 commit")
        return

    # 3. 更新版本号（假设有 version.py）
    print("  更新版本号...")

    # 4. 生成 changelog
    auto_changelog()

    # 5. 提交版本更新
    subprocess.run(["git", "add", "."])
    subprocess.run(["git", "commit", "-m", f"chore: release {version}"])

    # 6. 打标签
    subprocess.run(["git", "tag", "-a", version, "-m", f"版本 {version}"])

    # 7. 推送
    subprocess.run(["git", "push", "origin", branch])
    subprocess.run(["git", "push", "origin", "--tags"])

    print(f"✅ {version} 发布完成！")

def git_stats(since="2026-01-01"):
    """统计代码库健康状况"""
    stats = {}

    # 提交数
    r = subprocess.run(["git", "rev-list", "--count", "HEAD", "--since", since],
                      capture_output=True, text=True)
    stats["commits"] = int(r.stdout.strip())

    # 作者数
    r = subprocess.run(["git", "shortlog", "-sn", "--since", since],
                      capture_output=True, text=True)
    lines = [l for l in r.stdout.strip().split("\n") if l.strip()]
    stats["authors"] = len(lines)

    # 分支数
    r = subprocess.run(["git", "branch"], capture_output=True, text=True)
    stats["branches"] = len(r.stdout.strip().split("\n"))

    # 标签数
    r = subprocess.run(["git", "tag"], capture_output=True, text=True)
    stats["tags"] = len([t for t in r.stdout.strip().split("\n") if t])

    # 仓库大小
    r = subprocess.run(["git", "count-objects", "-vH"], capture_output=True, text=True)
    for line in r.stdout.split("\n"):
        if "size-pack" in line:
            stats["size"] = line.split(":")[-1].strip()

    print("📊 Git 仓库统计")
    print("="*40)
    for k, v in stats.items():
        print(f"  {k:10}: {v}")
    return stats

🎯 15. 面试高频题

1. Git 和 SVN 的区别？

Git 分布式，每个本地有完整仓库；SVN 集中式。Git 分支轻量，svn 分支是目录拷贝

2. merge 和 rebase 区别？

merge 保留分叉历史，rebase 线性化。rebase 修改 commit hash，不要在公共分支用

3. 如何撤销已经推送的 commit？

用 git revert <hash> 生成反提交（安全）。不用 reset（会改写历史）

4. git reset --soft/mixed/hard 区别？

soft 只改 HEAD / mixed 改 HEAD+暂存区 / hard 全改（含工作区，⚠ 危险）

5. 如何解决冲突？

编辑文件删除 <<<===>>> 标记 → git add → git commit。或 git mergetool

6. 如何找回误删的 commit？

git reflog 找到 hash → git reset --hard <hash> 或 git cherry-pick
