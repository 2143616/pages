2 创建数据集

2.1 Alpaca 格式

LLaMA-Factory 使用标准的 Alpaca 格式（JSON 数组），每条数据包含三个字段：

字段说明必填
instruction用户的指令/问题✅
input可选的输入上下文（可留空）❌
output期望的模型回答✅
金融问答示例JSON
{
  "instruction": "什么是复利？",
  "input": "",
  "output": "复利是指利息加入本金后再计算利息，通俗说就是利滚利。比如你存100元，年息10%，第一年得10元利息，第二年按110元算利息得11元，逐年递增。"
}

2.2 孙悟空风格数据集

让模型用孙悟空的语气讲解理财知识——instruction 是金融问题，output 用孙悟空的口吻回答。

特征示例
自称"俺老孙"、"俺"
语气词"哈哈"、"嘿嘿"、"呔！"、"唔"
西游记典故金箍棒、筋斗云、花果山、蟠桃园、紧箍咒
语言风格半文半白、豪爽自信、类比生动
💬 问题:什么是复利？
🤖 孙悟空风格回答:嘿嘿，复利？好比俺老孙当年在东海龙宫作客时遇到的石猴变仙的故事！当初那石猴要修行成仙，有师傅传授了它多少法术——这便是本金；再有那师父耐心地教他、鼓励他——这便是利息。时间一长，这利息又会长出利息来，就像那蟠桃园里的桃子越熟越大一样——这就是复利！

3 训练配置文件详解

train_wukong_qlora.yamlYAML
# 模型配置
model_name_or_path: models/qwen2.5-1.5b   # 基础模型路径
trust_remote_code: true

# 训练方法 —— 三个最重要的参数
stage: sft                                 # 【必须】SFT = 监督微调
finetuning_type: lora                      # lora | full | ia3
quantization_bit: 4                        # 4-bit QLoRA
use_unsloth: true                          # Unsloth 加速

# LoRA 参数
lora_target: all
lora_rank: 8                               # 8 是效果和效率的最佳平衡
lora_alpha: 16
lora_dropout: 0.05

# 数据集
dataset: wukong_finance
dataset_dir: data
template: qwen
cutoff_len: 1024

# 训练参数
do_train: true                             # 【必须】执行训练
per_device_train_batch_size: 1
gradient_accumulation_steps: 4
learning_rate: 2.0e-4
num_train_epochs: 5.0
lr_scheduler_type: cosine
warmup_ratio: 0.05

# 保存
output_dir: checkpoint-wukong
logging_steps: 5
save_strategy: epoch
save_total_limit: 3

# 精度 —— 关键！取决于 GPU 架构
bf16: false                                # Turing (RTX 20xx) 不支持
fp16: true                                 # Turing 用 FP16
pure_bf16: false
flash_attn: auto

关键参数速查

参数推荐值说明
stagesft不写的话只跑评估不训练！
do_traintrue默认 false！不写就不会训练
bf16取决于 GPUAmpere (RTX 30xx) 用 true；Turing (RTX 20xx) 用 false
quantization_device_map不要写！训练时设置会报错，仅推理可用
🚨 最常见的坑：stage: sft 和 do_train: true 缺一不可。少了任何一个，LLaMA-Factory 会只跑评估就退出（exit code 0），看起来成功了但没有训练！

5 导出 GGUF + Ollama 部署

训练完成后 LoRA adapter 只有 36MB，需要合并到基础模型再转 GGUF。流程：

1合并 LoRA
→
2转 GGUF
→
3量化 Q4_K_M
→
4Ollama 创建
→
5推理测试

5.1 导出配置

export_wukong.yamlYAML
model_name_or_path: models/qwen2.5-1.5b
adapter_name_or_path: checkpoint-wukong
finetuning_type: lora
template: qwen
export_dir: gguf-export-wukong
export_size: 2

5.2 完整导出命令

bashSHELL
# Step 1: 合并 LoRA
llamafactory-cli export config/export_wukong.yaml

# Step 2: 安装 llama.cpp 转换工具
git clone --depth 1 https://github.com/ggml-org/llama.cpp.git /tmp/llama.cpp
pip install -r /tmp/llama.cpp/requirements/requirements-convert_hf_to_gguf.txt

# Step 3: safetensors → GGUF (FP16)
python3 /tmp/llama.cpp/convert_hf_to_gguf.py gguf-export-wukong \
  --outfile gguf-export-wukong/model-f16.gguf --outtype f16

# Step 4: 编译 llama-quantize + 量化到 Q4_K_M
cd /tmp/llama.cpp && cmake -B build-cpu -DCMAKE_BUILD_TYPE=Release
cmake --build build-cpu --target llama-quantize -j$(nproc)
build-cpu/bin/llama-quantize \
  gguf-export-wukong/model-f16.gguf \
  gguf-export-wukong/model-q4_k_m.gguf Q4_K_M

# Step 5: 修复 Ollama 校验（关联 llama-quantize）
sudo ln -sf /tmp/llama.cpp/build-cpu/bin/llama-quantize \
  /usr/local/lib/ollama/llama-quantize

6.2 常见问题

Q: CUDA Out of Memory 怎么办？
减小 per_device_train_batch_size 到 1；减小 cutoff_len 到 512；启用 use_unsloth: true；换小模型（1.5B 代替 7B）；加环境变量 PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True
Q: Loss 下降但验证 Loss 上升？
过拟合。减少 epoch 到 3；增加数据量；增大 lora_dropout 到 0.1；减小 lora_rank 到 4。最佳 checkpoint 通常是验证 Loss 最低的那个，不一定是最后一个 epoch。
Q: Loss 完全不动（一直 2.3 左右）？
检查配置里是否有 stage: sft 和 do_train: true。少了任何一个模型会只跑评估不训练。
Q: 不同显存可以训练多大的模型？
6GB → 1.5B~3B | 8GB → 1.5B~3B ✅（本教程） | 12GB → 7B | 24GB → 7B~14B | 48GB+ → 32B~70B
