5 提交到本地仓库
Bash

Python
# 一行完成所有：先添加已修改的 index.html，再提交
$ cd /home/hyl/Desktop/资料/html && git add index.html && git commit -m "docs: 添加新文档到首页导航"
6 推送到 GitHub 远程仓库
Bash
# 本地分支名 不等于 远程分支名时：
$ git push origin HEAD:main
To https://github.com/2143616/pages.git
   226285d..debc8ab  HEAD -> main

# 同名分支时：
$ git push origin main
⚠️ 注意： 如果远程有本地没有的提交，会 reject。需要先 git pull origin main --rebase 再 push。

❌ 常见错误及解决

错误信息原因解决
fatal: The upstream branch does not match the name of your current branch 本地分支名和远程分支名不一致 git push origin HEAD:main
! [rejected] (fetch first) 远程有新 commit，本地没有 git pull origin main --rebase
→ 再 git push origin HEAD:main
! [rejected] (non-fast-forward) 本地和远程分叉了 git pull --rebase 或 git merge 后再推
error: RPC failed; curl 56 GnuTLS recv error 网络不稳定（常见国内） 使用代理：git config --global http.proxy socks5://127.0.0.1:7897

🎯 第 4 步：更新首页导航（重要）

🔑 关键点： 文件本身部署到 Vercel 后可以通过 URL 直接访问（如 /Python.html），
但如果 首页 index.html 没有链接，访客无法从主页找到新文档。

所以每次添加新文档后，需要同步更新 index.html：

流程
1. 编辑 index.html，添加新文档的 <a> 链接卡片
2. git add index.html
3. git commit -m "docs: 添加新文档到首页导航"
4. git push origin HEAD:main       ← 再次触发 Vercel 部署
5. 等待几秒，刷新网站 → 新链接出现在首页

🐍 Python 自动化脚本

也可以用 Python 一键提交并推送：

Python
import subprocess
import os

def git_deploy(commit_msg: str):
    """一键 add → commit → push"""
    os.chdir("/home/hyl/Desktop/资料/html")

    # 1. git add
    subprocess.run(["git", "add", "."])

    # 2. git commit
    r = subprocess.run(["git", "commit", "-m", commit_msg],
                       capture_output=True, text=True)
    print(r.stdout)

    # 3. git push
    r = subprocess.run(["git", "push", "origin", "HEAD:main"],
                       capture_output=True, text=True)
    print(r.stdout)

    print("✅ 已推送，Vercel 自动部署中...")

# 使用
# git_deploy("docs: 更新教学文档")
