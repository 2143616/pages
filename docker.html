1. Docker 概述

什么是 Docker？

Docker 是一个开源的容器化平台，它允许开发者将应用程序及其所有依赖打包到一个轻量级、可移植的容器中，然后在任何支持 Docker 的环境中运行。

核心理念：Build Once, Run Anywhere（一次构建，处处运行）

Docker vs 虚拟机

特性Docker 容器虚拟机
启动速度秒级分钟级
资源占用MB 级别GB 级别
隔离级别进程级隔离完全隔离
OS 支持共享宿主机内核独立完整 OS
磁盘占用镜像层共享，极小每个 VM 独立磁盘
迁移性镜像分发，极其轻量需要迁移整个 VM

Docker 架构

Docker 使用 客户端-服务端（C/S）架构：

  • Docker Client：用户通过 docker CLI 与 daemon 交互
  • Docker Daemon（dockerd）：后台服务，管理镜像、容器、网络、存储
  • Registry：镜像仓库，如 Docker Hub、Harbor、阿里云容器镜像服务
docker client  →  REST API  →  docker daemon  →  containerd  →  runc  →  容器进程

核心组件

  • containerd：工业级容器运行时，管理容器生命周期
  • runc：OCI 容器运行时规范的参考实现，实际创建容器
  • BuildKit：新一代镜像构建引擎（Docker 18.09+）
  • Docker Compose：多容器应用编排工具
  • Docker Swarm：Docker 原生集群管理（已被 K8s 超越但仍可用）

2. 安装与配置

Linux（以 Ubuntu 为例）

# 1. 卸载旧版本
sudo apt-get remove docker docker-engine docker.io containerd runc

# 2. 安装依赖
sudo apt-get update
sudo apt-get install ca-certificates curl gnupg

# 3. 添加官方 GPG Key
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# 4. 添加仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 5. 安装 Docker Engine
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 6. 验证安装
sudo docker run hello-world

# 7. 将当前用户加入 docker 组（免 sudo）
sudo usermod -aG docker $USER
newgrp docker  # 或重新登录

国内加速配置

# 编辑 /etc/docker/daemon.json
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://docker.m.daocloud.io"
  ],
  "log-driver": "json-file",
  "log-opts": { "max-size": "10m", "max-file": "3" },
  "storage-driver": "overlay2"
}

# 重启使配置生效
sudo systemctl daemon-reload
sudo systemctl restart docker

macOS / Windows

推荐使用 Docker Desktop：下载安装包一键安装，内置 Docker Engine、Kubernetes、Docker Compose、GUI 管理界面。

rootless 模式（安全增强）

# 以非 root 用户运行 Docker 守护进程
dockerd-rootless-setuptool.sh install
# 设置环境变量
export PATH=/usr/bin:$PATH

3. 核心概念

镜像（Image）

镜像是容器的只读模板，包含运行应用所需的完整文件系统。镜像由多个只读层（layer）叠加组成，每层代表一条 Dockerfile 指令。

容器（Container）

容器是镜像的运行实例。它在镜像的只读层之上添加一个可写层（Container Layer），所有运行时变更（文件增删改）都写入这层。

仓库（Registry / Repository）

  • Registry：存储和分发镜像的服务，如 Docker Hub
  • Repository：Registry 中一个镜像的集合，如 library/nginx
  • Tag：镜像版本标签，如 nginx:1.25-alpine

镜像完整路径格式：[registry/][namespace/]repository[:tag]

docker.io/library/nginx:latest           # 完整写法
nginx:latest                              # 简写（默认 docker.io/library）
my-registry.com:5000/myapp:v1.0           # 私有仓库

层（Layer）与写时复制（CoW）

  • 镜像由多个只读层堆叠，使用 UnionFS（Overlay2）合并为一个视图
  • 容器可写层使用 Copy-on-Write：修改只读层的文件时，先将文件复制到可写层再修改
  • 层共享机制：同一宿主机上的多个容器可共享相同的底层镜像层，极大节省磁盘

4. 镜像管理

查找镜像

docker search nginx                    # 搜索 Docker Hub
docker search --limit 10 alpine        # 限制结果数
docker search --filter=stars=100 nginx # 按星数过滤

拉取镜像

docker pull nginx:1.25                 # 指定版本
docker pull nginx:latest               # 最新版
docker pull node:20-alpine             # Alpine 精简版
docker pull --platform linux/arm64 nginx  # 指定架构

查看镜像

docker images                          # 列出本地镜像
docker images -a                       # 包含中间层
docker images -f dangling=true         # 显示悬空镜像（无 tag）
docker image ls --format "table {{.Repository}}\t{{.Tag}}\t{{.Size}}"
docker image history nginx:latest      # 查看镜像构建历史（层）
docker image inspect nginx:latest      # 查看镜像详细信息（JSON）
docker image inspect -f '{{.Config.Env}}' nginx  # 格式化提取字段

删除镜像

docker rmi nginx:latest                # 删除指定镜像
docker rmi $(docker images -q)         # 删除所有镜像（危险！）
docker image prune                     # 删除所有悬空镜像
docker image prune -a                  # 删除所有未使用的镜像
docker image prune -a --filter "until=24h"  # 删除24小时前的未用镜像

导出与导入

docker save -o nginx.tar nginx:latest  # 导出镜像为 tar 文件
docker load -i nginx.tar               # 从 tar 导入镜像
# 带压缩
docker save nginx:latest | gzip > nginx.tar.gz
gunzip -c nginx.tar.gz | docker load

镜像标签

docker tag nginx:latest myrepo/nginx:v1.0    # 创建新标签
docker tag SOURCE_IMAGE[:TAG] TARGET[:TAG]   # 实际上是创建别名（引用）

推送镜像

docker login                           # 登录 Docker Hub
docker login my-registry.com:5000      # 登录私有仓库
docker push myrepo/nginx:v1.0          # 推送镜像
docker logout                          # 登出

创建与运行

# 基础运行
docker run nginx:latest

# 常用组合

# -d: 后台运行  --name: 命名  -p 宿主机:容器端口映射

# 完整示例
docker run -d \
  --name my-nginx \
  --restart always \
  -p 8080:80 \
  -p 8443:443 \
  -v /home/user/html:/usr/share/nginx/html:ro \
  -v nginx-logs:/var/log/nginx \
  -e NGINX_HOST=example.com \
  --memory 256m \
  --cpus 1.0 \
  --network my-net \
  nginx:1.25-alpine

重启策略

策略行为
no默认，不自动重启
always总是重启（Docker 启动时也会重启）
unless-stopped除非手动停止，否则重启（推荐）
on-failure[:N]仅非零退出时重启，可限制次数

生命周期管理

docker ps                              # 列出运行中的容器
docker ps -a                           # 列出所有容器（含停止的）
docker ps -q                           # 只显示容器 ID
docker ps --format "{{.Names}}: {{.Status}}"

docker start my-container              # 启动已停止的容器
docker stop my-container               # 优雅停止（SIGTERM → SIGKILL）
docker stop -t 30 my-container         # 等待30秒后强杀
docker restart my-container            # 重启
docker pause my-container              # 暂停进程（SIGSTOP）
docker unpause my-container            # 恢复进程（SIGCONT）
docker kill my-container               # 立即强杀（SIGKILL）
docker rm my-container                 # 删除已停止的容器
docker rm -f my-container              # 强制删除（运行中也删）
docker rm $(docker ps -aq)             # 删除所有容器（危险）
docker container prune                 # 删除所有已停止容器
docker container prune --filter "until=24h"

交互式操作

docker exec -it my-container bash      # 进入运行中的容器
docker exec my-container ls /app       # 执行单条命令
docker attach my-container             # 附加到容器主进程（慎用）
docker logs my-container               # 查看日志
docker logs -f my-container            # 实时跟踪日志
docker logs --tail 100 my-container    # 最后100行
docker logs --since 1h my-container    # 最近1小时日志

容器内文件操作

docker cp my-container:/app/log.txt ./     # 从容器复制到宿主机
docker cp ./config.yaml my-container:/app/  # 从宿主机复制到容器

查看容器详情

docker inspect my-container            # 完整配置（JSON）
docker inspect -f '{{.NetworkSettings.IPAddress}}' my-container  # 提取IP
docker top my-container                # 查看容器内进程
docker stats                           # 实时查看所有容器资源使用
docker stats my-container              # 指定容器
docker port my-container               # 查看端口映射
docker diff my-container               # 查看容器文件系统变更

commit（不推荐但有用）

docker commit my-container myimage:v2   # 将容器保存为新镜像
docker commit -m "added config" -a "author" my-container myimage:v2
注意：docker commit 会导致镜像不可复现、层膨胀。推荐用 Dockerfile 构建。

6. Dockerfile 详解

Dockerfile 是构建镜像的声明式蓝图，每条指令创建一个新的镜像层。

6.1 常用指令

FROM

FROM node:20-alpine                     # 基础镜像
FROM node:20-alpine AS builder          # 多阶段构建，命名阶段
FROM scratch                            # 空白镜像（极简/静态编译程序）

RUN

# shell 形式（默认 /bin/sh -c）
RUN apt-get update && apt-get install -y curl

# exec 形式（不启动 shell，需要手动处理变量展开）
RUN ["apt-get", "update"]

# 多层合并减少层数和镜像体积
RUN apt-get update && \
    apt-get install -y --no-install-recommends \
        curl \
        wget \
        ca-certificates && \
    rm -rf /var/lib/apt/lists/*

COPY vs ADD

# COPY：简单复制（推荐）
COPY package.json yarn.lock /app/
COPY . /app/

# ADD：额外支持 tar 自动解压和 URL 下载
ADD archive.tar.gz /app/
ADD https://example.com/file.txt /app/

# COPY 的 --chown 选项
COPY --chown=node:node . /app/
最佳实践：优先使用 COPY，只有需要自动解压才用 ADD。

WORKDIR

WORKDIR /app                            # 后续指令的相对路径基准
# 相当于 RUN mkdir -p /app && cd /app

ENV

ENV NODE_ENV=production \
    APP_PORT=3000

# 运行时还可被 -e 覆盖
docker run -e NODE_ENV=development myapp

ARG

ARG VERSION=1.0.0                      # 构建时变量，默认值
RUN echo "Building version $VERSION"
# 构建时传参：docker build --build-arg VERSION=2.0.0 .

EXPOSE

EXPOSE 3000                             # 声明容器监听端口（文档性质）
# 实际映射需要 run -p 或 compose ports

CMD vs ENTRYPOINT

指令用途是否可被覆盖
CMD默认启动命令可被 docker run 末尾参数完全覆盖
ENTRYPOINT固定入口程序不会被覆盖，docker run 参数作为其参数
# 仅 CMD（最常用）
CMD ["node", "server.js"]

# ENTRYPOINT + CMD 组合（CMD 作为默认参数）
ENTRYPOINT ["python", "app.py"]
CMD ["--port", "8080"]
# docker run myapp --host 0.0.0.0  →  python app.py --host 0.0.0.0

# shell form vs exec form
CMD echo "hello"                        # shell form：启动 /bin/sh -c
CMD ["echo", "hello"]                   # exec form：直接执行（推荐，信号传递正确）

其他指令

# LABEL：元数据标签
LABEL maintainer="user@example.com" \
      version="1.0" \
      description="My Docker application"

# USER：切换运行用户
RUN useradd -m -s /bin/bash appuser
USER appuser

# HEALTHCHECK：健康检查
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD curl -f http://localhost:3000/health || exit 1

# STOPSIGNAL：停止信号
STOPSIGNAL SIGTERM

# SHELL：覆盖默认 shell
SHELL ["/bin/bash", "-c"]

6.2 多阶段构建（Multi-stage Build）

在一个 Dockerfile 中使用多个 FROM，只将最终需要的产物复制到最终镜像，中间构建工具不进入最终镜像。

# ============= 阶段1：构建 =============
FROM node:20-alpine AS builder
WORKDIR /app
COPY package.json yarn.lock ./
RUN yarn install --frozen-lockfile --production=false
COPY . .
RUN yarn build

# ============= 阶段2：运行 =============
FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production

# 从构建阶段只复制产物
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package.json ./

EXPOSE 3000
USER node
CMD ["node", "dist/server.js"]
# Go 静态编译：最终镜像可以是从 scratch
FROM golang:1.21-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -o /app/server .

FROM scratch
COPY --from=builder /etc/ssl/certs/ca-certificates.crt /etc/ssl/certs/
COPY --from=builder /app/server /server
ENTRYPOINT ["/server"]

6.3 Dockerfile 最佳实践

  1. 使用 .dockerignore——排除 node_modules、.git、构建产物
  2. 一个容器一个进程——不要在一个容器里跑多个服务
  3. 最小化层数——合并 RUN 指令，清理缓存
  4. 利用构建缓存——把变化少的文件（package.json）放前面
  5. 使用特定版本 Tag——不要用 latest
  6. 以非 root 用户运行——USER 指令
  7. 镜像体积最小化——Alpine 基础镜像 + 多阶段构建
  8. 标签与元数据——添加 LABEL
# .dockerignore 示例
node_modules
.git
.gitignore
*.md
dist
.env
.env.local
Dockerfile
docker-compose.yml
.vscode
.idea
coverage
*.log

BuildKit 高级构建

# 启用 BuildKit（推荐，Docker 23+ 默认启用）
export DOCKER_BUILDKIT=1

# --mount=type=cache：缓存依赖，跨构建复用
RUN --mount=type=cache,target=/root/.npm \
    npm install

# --mount=type=secret：安全传递构建密钥
RUN --mount=type=secret,id=aws_credentials \
    AWS_SHARED_CREDENTIALS_FILE=/run/secrets/aws_credentials \
    aws s3 cp ...

# --mount=type=ssh：传递 SSH 密钥
RUN --mount=type=ssh \
    git clone git@github.com:user/private-repo.git

7. 数据管理

容器是无状态的——删除容器后容器可写层的数据丢失。持久化数据有3种方式：

7.1 Volumes（推荐）

由 Docker 管理的宿主机目录（/var/lib/docker/volumes/），完全由 Docker 控制。

# 创建卷
docker volume create my-vol

# 查看
docker volume ls
docker volume inspect my-vol

# 使用
docker run -v my-vol:/app/data nginx   # -v 卷名:容器路径
docker run --mount source=my-vol,target=/app/data nginx  # --mount（更明确）

# 删除
docker volume rm my-vol
docker volume prune                    # 删除所有未被使用的卷

7.2 Bind Mounts（绑定挂载）

将宿主机上的任意目录/文件挂载到容器中，用于开发热重载或共享配置文件。

docker run -v /host/path:/container/path nginx
docker run -v /host/path:/container/path:ro nginx      # 只读挂载
docker run --mount type=bind,source=/host/path,target=/container/path nginx

7.3 tmpfs

存储在宿主机内存中，不写入磁盘，容器停止即消失。适合临时敏感数据。

docker run --tmpfs /app/tmp nginx
docker run --mount type=tmpfs,destination=/app/tmp,tmpfs-size=64m nginx

三种方式对比

方式位置持久化跨容器共享性能
Volume/var/lib/docker/volumes/是是好
Bind Mount任意宿主机路径是是取决于底层FS
tmpfs内存否否极快

8. 网络管理

8.1 网络驱动

驱动说明使用场景
bridge默认网络驱动，容器通过虚拟网桥通信单机容器互联
host直接使用宿主机网络栈，无隔离高性能需求或无需隔离
overlay跨多宿主机的容器通信（Swarm）集群/分布式应用
macvlan容器拥有独立 MAC 地址，直接暴露在物理网络需要容器在局域网可见
none禁用网络无网络需求

8.2 自定义网络

# 创建 bridge 网络
docker network create my-net

# 创建时指定参数
docker network create \
  --driver bridge \
  --subnet 172.20.0.0/16 \
  --gateway 172.20.0.1 \
  --ip-range 172.20.0.0/24 \
  my-net

# 查看
docker network ls
docker network inspect my-net

# 连接/断开容器
docker network connect my-net my-container
docker network disconnect my-net my-container

# 删除
docker network rm my-net
docker network prune                    # 删除未使用的网络
关键特性：同一自定义 bridge 网络中的容器可通过容器名互相访问，Docker 内置 DNS 自动解析。
# 示例：nginx + node 通过容器名通信
docker network create app-net

docker run -d --network app-net --name backend my-node-app
docker run -d --network app-net --name frontend -p 80:80 nginx
# frontend 可以通过 http://backend:3000 访问 backend

9. Docker Compose

Docker Compose 用于定义和运行多容器应用。通过一个 docker-compose.yml 文件描述所有服务、网络、卷。

9.1 Compose 文件示例

# docker-compose.yml
version: "3.9"

services:

  # PostgreSQL 数据库
  db:
    image: postgres:16-alpine
    container_name: app-db
    restart: unless-stopped
    environment:
      POSTGRES_USER: ${DB_USER:-appuser}
      POSTGRES_PASSWORD: ${DB_PASSWORD:-secret}
      POSTGRES_DB: myapp
    volumes:
      - pgdata:/var/lib/postgresql/data
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql  # 初始化脚本
    ports:
      - "5432:5432"
    networks:
      - backend
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U appuser -d myapp"]
      interval: 10s
      timeout: 5s
      retries: 5

  # Redis 缓存
  redis:
    image: redis:7-alpine
    container_name: app-redis
    restart: unless-stopped
    volumes:
      - redisdata:/data
    networks:
      - backend
    command: redis-server --appendonly yes --requirepass ${REDIS_PASSWORD}

  # 后端 API
  api:
    build:
      context: ./backend
      dockerfile: Dockerfile
      args:
        APP_VERSION: ${APP_VERSION:-1.0.0}
    image: myapp-api:latest
    container_name: app-api
    restart: unless-stopped
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_started
    environment:
      DATABASE_URL: postgresql://${DB_USER}:${DB_PASSWORD}@db:5432/myapp
      REDIS_URL: redis://:${REDIS_PASSWORD}@redis:6379
      NODE_ENV: production
    ports:
      - "3000:3000"
    volumes:
      - ./backend/uploads:/app/uploads
    networks:
      - backend

          cpus: "0.5"
          memory: 256M

      - ./certs:/etc/nginx/certs:ro

10. 安全最佳实践

镜像安全

  1. 使用可信基础镜像——优先官方镜像 + 固定 TAG（不要 latest）
  2. 定期扫描漏洞——docker scout quickview myimage 或 Trivy
  3. 最小化攻击面——Alpine/Distroless 基础镜像 + 多阶段构建
  4. 使用特定版本号——可复现 + 可审计
# Docker Scout 漏洞扫描
docker scout quickview nginx:latest
docker scout cves nginx:latest

# Trivy 扫描（需安装）
trivy image myapp:latest

运行时安全

# 以非 root 用户运行
docker run -u 1000:1000 myapp

# 只读根文件系统
docker run --read-only --tmpfs /tmp myapp

# 限制内核能力（capabilities）
docker run --cap-drop ALL --cap-add NET_BIND_SERVICE myapp

# 禁止特权模式
# 不要使用 --privileged（除非绝对必要）

# 限制系统调用（seccomp）
docker run --security-opt seccomp=my-seccomp.json myapp

# AppArmor / SELinux
docker run --security-opt apparmor=my-profile myapp

# 禁用容器间通信
docker network create --opt com.docker.network.bridge.enable_icc=false secure-net
docker run --network secure-net myapp

密钥管理

# Docker Secret（Swarm 模式）
echo "mysecretpassword" | docker secret create db_password -
docker service create --secret db_password myapp

# BuildKit secret（构建时安全传递）
RUN --mount=type=secret,id=github_token \
    GITHUB_TOKEN=$(cat /run/secrets/github_token) \
    npm install

# 不要这样做！
# ENV API_KEY=secret123          ← 密钥会留在镜像层中
# COPY .env /app/.env            ← 密钥会留在镜像文件系统中

12. 资源限制

# CPU 限制
docker run --cpus 1.5 myapp                      # 最多1.5个CPU核心
docker run --cpuset-cpus 0-2 myapp               # 绑定到CPU 0,1,2
docker run --cpu-shares 512 myapp                # 相对权重（默认1024）

# 内存限制
docker run -m 512m myapp                         # 最大512M内存
docker run -m 512m --memory-swap 1g myapp        # 512M内存 + 512M交换
docker run -m 512m --memory-swap 512m myapp      # 禁用交换（等于内存值）
docker run --memory-reservation 256m myapp       # 软限制

# 磁盘 I/O 限制
docker run --blkio-weight 500 myapp              # 相对权重（10-1000）
docker run --device-read-bps /dev/sda:1mb myapp  # 读速率限制
docker run --device-write-bps /dev/sda:1mb myapp # 写速率限制
docker run --device-read-iops /dev/sda:100 myapp # 读 IOPS 限制
docker run --device-write-iops /dev/sda:100 myapp

# PID 限制（防止 fork 炸弹）
docker run --pids-limit 100 myapp

# ulimit 限制
docker run --ulimit nofile=65535:65535 myapp     # 最大打开文件数
docker run --ulimit nproc=1024:2048 myapp        # 最大进程数

13. CI/CD 集成

GitHub Actions 示例

# .github/workflows/docker.yml
name: Build and Push Docker Image

on:
  push:
    branches: [main]
    tags: ['v*']

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write

    steps:
      - uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login to Container Registry
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Extract metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}
          tags: |
            type=ref,event=branch
            type=ref,event=pr
            type=semver,pattern={{version}}
            type=semver,pattern={{major}}.{{minor}}

      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

15. 常见问题与排错

容器启动后立即退出

# 原因：主进程（CMD）执行完就结束了
# 检查日志
docker logs my-container

# 常见原因：
# 1. 脚本报错退出
# 2. 缺少前台进程（如 CMD ["bash"] 不带 -it 会退出）
# 3. 端口冲突

容器内无法访问网络

# DNS 问题
docker run --dns 8.8.8.8 myapp
# 或修改 /etc/docker/daemon.json 添加 dns 配置

# 代理问题
docker run -e HTTP_PROXY=http://proxy:8080 myapp

权限拒绝（Permission Denied）

# 挂载目录的权限问题
# 容器内的 UID 和宿主机不一致
docker run -u $(id -u):$(id -g) -v /host/path:/data myapp

# 或 Dockerfile 中创建对应 UID 的用户：
RUN useradd -m -u 1000 appuser && chown -R appuser:appuser /app

磁盘空间不足

# 检查占用
docker system df
docker system df -v

# 清理
docker system prune -a --volumes

# 检查 overlay2 占用
sudo du -sh /var/lib/docker/

时区不正确

# 挂载宿主机时区
docker run -v /etc/localtime:/etc/localtime:ro -v /etc/timezone:/etc/timezone:ro myapp

# 或在 Dockerfile 中设置
ENV TZ=Asia/Shanghai
RUN ln -snf /usr/share/zoneinfo/$TZ /etc/localtime && echo $TZ > /etc/timezone

端口已被占用

# 查看端口占用
sudo lsof -i :8080
sudo ss -tlnp | grep 8080

# 解决方法：
# 1. 换个端口 -p 8081:80
# 2. 停止占用进程
# 3. 用 host 网络模式（不推荐）

  • GitHub Container Registry（ghcr.io）：与 GitHub 生态集成
  • 阿里云容器镜像服务（ACR）：国内加速

CI/CD 与 DevOps 工具

  • Jenkins + Docker Pipeline
  • GitHub Actions / GitLab CI

  • Drone CI：轻量级 CI，原生容器支持
