📖 核心概念科普

在深入项目之前，先理解几个关键概念——这对看懂后续内容至关重要。

🔄

模仿学习 (IL)

人先做一遍演示（遥操作），记录"看到什么→做了什么"的数据对，然后训练神经网络学会像人一样动作。好比"看着师傅做一遍，然后自己照着做"。

🏋️

强化学习 (RL)

让机器人在仿真环境中自己试错——做对了给奖励，做错了扣分。经过几百万次尝试，自然找到最优策略。Sim-to-Real 就是把仿真中学到的策略搬到真机。

👁️

VLA（视觉-语言-动作）

融合视觉理解 + 语言指令 + 动作输出的多模态大模型。你说"帮我拿杯子"，它看到杯子、理解你的意图、规划抓取轨迹并执行。Pi0、SmolVLA 都是 VLA。

🎮

Sim-to-Real（仿真到真机）

在虚拟环境（Isaac Sim、ManiSkill3）中训练策略，再把训练好的模型直接部署到真实机器人上。优点是仿真中可以跑几百万次尝试而不损坏真机。

📦

LeRobotDataset

LeRobot 定义的标准数据集格式：MP4 存摄像头视频，Parquet 存关节角度/动作/状态。统一格式让不同团队的机器人数据可以互相训练。

🤝

遥操作 (Teleoperation)

人操作 Leader 臂（主臂），Follower 臂（从臂）实时跟随。这是最直观的数据采集方式——人类通过"手把手"教学，把技能传授给机器人。

🤯 关键洞察

LeRobot 的核心价值在于标准化：不管你是用 SO-101 还是 Unitree G1 人形机器人，数据接口、训练脚本、评估流程都一致。这意味着 SO-101 上训练的策略，迁移到其他平台时只需改机器人配置。这种"硬件无关"设计大幅降低了 AI 机器人的入门门槛。

🔥 已落地的 AI 结合项目

以下是从 GitHub 和社区中收集的真实项目，每个都有明确的 AI 技术点。标⭐数为 GitHub Stars，反映项目热度。

📦 Squint — 快速视觉强化学习 ⭐ 80 RL Sim-to-Real PyTorch

基于 PyTorch 的快速视觉强化学习框架，在 ManiSkill3 仿真器中训练策略后直接迁移到 SO-101 真机。

🔬 技术拆解：该项目展示了 RL 领域最难的"Sim-to-Real 迁移"——仿真中的像素和物理与现实世界存在差异（Sim-to-Real Gap）。Squint 通过领域随机化（在仿真中随机改变光照、纹理、物理参数）让模型学会不依赖特定环境特征，从而在真机上也能正常工作。训练完成后，模型通过摄像头看到真机场景，输出关节角度动作，实现"看→决策→动作"的闭环。
🏔️ isaac_so_arm101 — NVIDIA Isaac Lab 集成 ⭐ 280 NVIDIA RL 仿真

在 NVIDIA Isaac Sim + Isaac Lab 中为 SO-ARM100/101 搭建完整仿真环境，支持 RL 训练和数据采集。

🔬 技术拆解：Isaac Lab 是 NVIDIA 的机器人 RL 框架，底层使用 PhysX 物理引擎提供高保真仿真。该项目为 SO-101 构建了 URDF 模型、碰撞体、关节限位、相机参数等完整数字孪生。用户可以在仿真中运行 PPO / SAC / TDMPC 等 RL 算法，训练策略后再零样本迁移到真机。配套中文教程系列是中文社区最完整的 SO-101 + Isaac Sim 学习资料。
🎮 Isaac Lab Teleop — 仿真数据采集 ⭐ 63 数据采集 Isaac Lab

在 Isaac Lab 中搭建 SO-101 仿真遥操作环境，采集示教演示数据用于模仿学习。

🔬 技术拆解：真机采集数据容易受硬件磨损、电机发热、布线限制。该项目在 Isaac Lab 中实现了仿真遥操作——你用操纵杆或键盘在虚拟环境中控制机械臂，系统自动录制"视觉观察→关节动作"的数据对。这些数据可以用来训练模仿学习模型，而不会占用真机。同时，仿真数据可以低成本大量生成，大幅扩展训练数据规模。
🇨🇳 EmbodiedFX — 全流程中文教程 ⭐ 7 入门 教程 中英双语

从 SO-101 组装、数据采集、模型微调到真机推理的完整教程，中英双语。

🔬 技术拆解：这是 SO-101 入门的最佳起点。教程涵盖：①硬件组装与校准 ②遥操作数据采集（录制 50 个 episodes 的抓取演示）③用采集的数据微调 ACT 模型（Action Chunking Transformer）④将训练好的模型部署到真机让机械臂自主执行任务。整个过程不涉及复杂理论，适合第一次接触机器人 AI 的开发者。
🎤 语音控制 + SmolVLA ⭐ 3 VLA 语音 多模态

语音指令控制 SO-101 机械臂，使用 SmolVLA 视觉-语言-动作模型理解自然语言并生成动作。

🔬 技术拆解：这是 VLA（Vision-Language-Action）模型在低成本机器人上的典型应用。工作流：①语音输入（麦克风）→ ②ASR 语音识别转文字 → ③SmolVLA 模型接收"摄像头图像 + 文字指令" → ④输出关节动作序列。SmolVLA 是 LeRobot 中体积最小、速度最快的 VLA 模型，它本质上是把视觉语言模型（VLM）的推理能力与动作生成头联合微调的结果，让模型学会"看到场景 → 理解指令 → 规划动作"的端到端映射。
👁️ 视觉伺服抓取系统 ⭐ 3 计算机视觉 YOLO-World 闭环控制

SO-101 六轴机械臂视觉伺服抓取系统，YOLO-World 开放词汇检测 + PySide6 GUI。

🔬 技术拆解："视觉伺服"（Visual Servoing）指的是视觉反馈闭环控制——摄像头持续拍摄目标物体，实时计算机械臂末端与目标的偏差，控制算法不断调整关节角度直到抓取成功。与"看一眼然后盲抓"不同，伺服控制可以在目标移动时动态跟踪。该项目使用 YOLO-World（开放词汇目标检测模型）识别任意物体——你不用事先训练检测器，直接在 GUI 中输入"red cube"它就能找到。
🧦 Claude Vision + 袜子分拣 🌟 新 LLM + 机器人 Claude Vision

用 Claude Vision 视觉识别袜子颜色/种类，SO-101 执行分拣操作。展示了 LLM 视觉能力与低成本机器人的结合。

🔬 技术拆解：这是一个"大语言模型作为机器人'大脑'"的典型架构：①Claude Vision 拍摄当前场景并理解（"左边是红袜子，右边是蓝袜子"）②LLM 根据识别结果做出决策（"红袜子放到 A 区"）③将决策传给 SO-101 执行抓取和放置。与传统端到端模型不同，LLM 充当了符号推理层，可以处理开放词汇和复杂逻辑判断，而不需要为每种新物体重新训练。
☁️ AWS IoT 云集成 ⭐ 1 IoT AWS MQTT

通过 AWS IoT Greengrass v2 将 SO-101 接入云端，伺服遥测数据通过 MQTT 实时流式传输。

🔬 技术拆解：在工业场景中，远程监控机器人状态是一个刚需。该项目在 SO-101 上运行 AWS Greengrass 边缘运行时，将关节角度、电流、温度等遥测数据通过 MQTT 协议持续发送到 AWS 云。用户可以在云端仪表盘实时查看机械臂状态，也可以从云下发指令。这种边缘-云架构是工业 4.0 中机器人物联网（IoRT）的缩影。
🎓 ETH — 强化学习抓取课程项目 ⭐ 1 RL Squint ETH Zurich

ETH 机器人学习课程（Spring 2026）项目，基于 Squint + SO-101 实现视觉强化学习抓取。

🔬 技术拆解：顶尖大学课程项目，展示了 SO-101 进入学术教育领域的能力。项目基于 Squint（前面的第 1 个项目）搭建自定义 RL 环境、设计奖励函数、部署策略到真机。评估场景分三个难度：单物体抓取 → 分割场景（区分多个物体）→ 4 立方体高密度场景。这种渐进式评估是 RL 研究的标准方法论——先验证基础能力，再增加复杂度。
🌐 Web 轨迹规划器 ⭐ 2 Web 3D 可视化 轨迹规划

浏览器中 3D 可视化 SO-101 + 积木块场景，通过点击定义路径点，自动生成机器人轨迹。

🔬 技术拆解：这本质上是一个运动规划前端：用户在 3D 场景中点击设置机械臂需要经过的路径点（waypoints），系统自动执行逆运动学（IK）求解和插值，生成平滑的关节轨迹。逆运动学解决"末端想要到某个位置，每个关节应该转多少度"的问题，是机器人学中最经典的问题之一。Web 端实现让教学演示无需安装任何软件。
🔧 lododo-arm — ROS2 智能机械臂系统 ⭐ 12 ROS2 Humble

基于 ROS2 Humble 的完整机械臂系统，将 SO-101 接入 ROS2 生态，支持标准消息通信和组件复用。

🔬 技术拆解：ROS2（Robot Operating System 2）是机器人领域的"Linux"——一套标准的分布式通信框架。该项目为 SO-101 实现了 ROS2 驱动节点，发布 /joint_states（关节状态）、订阅 /joint_commands（关节指令），使 SO-101 可以与 ROS2 生态中的导航、SLAM、MoveIt2 运动规划等数千个已有包无缝配合。这对于教学和研究来说意义重大，学生可以直接使用 ROS2 的标准工具链操作 SO-101。

🚀 快速上手：从零到第一次自主抓取

以下是推荐的学习路径，从硬件组装到 AI 模型部署，逐步深入。

🟢 阶段一：硬件准备与基础连接

# 1. 安装 LeRobot
pip install lerobot

# 2. 验证安装和可用机器人
lerobot-info

# 3. 连接 SO-101 并测试
python -c "
from lerobot.robots.so100 import SO100Robot
robot = SO100Robot()
robot.connect()
obs = robot.get_observation()
print('关节角度:', obs['joint_angles'])
robot.disconnect()
"

🟡 阶段二：数据采集

# 遥操作并录制数据
lerobot record \
  --robot-type so100 \
  --fps 30 \
  --episodes 50 \
  --root data/my_dataset

# 可视化采集的数据
lerobot visualize --dataset-path data/my_dataset

💡 数据采集要点

每个 episode 就是一次"从初始状态到任务完成"的完整演示。建议采集 30-50 个 episodes，涵盖不同的初始位置和物体摆放方式。数据质量直接影响模型表现——动作要平滑、轨迹要一致、失败重录。

🔵 阶段三：训练策略

# 训练 ACT 模型（模仿学习）
lerobot train \
  --policy.type act \
  --dataset.repo-id your-username/so101-pick-place \
  --training.training_steps 50000 \
  --training.output_dir ./outputs/act_policy

🤔 为什么要选 ACT？

ACT（Action Chunking Transformer） 是 LeRobot 中最成熟的模仿学习模型之一。它的核心思想是"一次预测未来 N 步动作"而不是"看一步动一步"，这样可以平滑动作轨迹、减少抖动。对于 SO-101 上常见的抓取和放置任务，ACT 通常是最佳起点。

🔴 阶段四：真机推理

# 将训练好的模型部署到 SO-101 真机
lerobot eval \
  --policy.path ./outputs/act_policy \
  --env.type real \
  --robot.type so100 \
  --eval.n_episodes 10

机器人现在应该能从摄像机看到目标物体→推理出动作序列→自主完成抓取任务了！

❓ 常见问题

初学者最常问的问题，这里一次性回答。

🤔 我不会机器人学，能上手吗？

可以。LeRobot 的设计目标就是降低门槛。你只需要基本的 Python 知识，逆运动学、控制理论等经典机器人学概念都被封装好了。从遥操作采集数据开始，不需要自己写控制算法。

💰 SO-101 全套多少钱？

SO-101 套件（含主臂+从臂）约 ¥2000 人民币左右。一台普通电脑（建议有 NVIDIA GPU）即可运行训练。相比工业机械臂动辄十几万元的成本，这是最经济的 AI 机器人实验平台。

🎯 我可以用 SO-101 做什么课题？

常见方向：①模仿学习（学人类演示做任务）②Sim-to-Real 迁移（仿真训练→真机部署）③VLA 模型评测（对比各类 VLA 在小规模场景的表现）④人机交互（语音/手势控制机器人）⑤机器人数据集构建（采集高质量开源数据）⑥ROS2 集成开发。

💻 需要什么电脑配置？

数据采集：任意电脑就行。训练 ACT/Diffusion：建议 NVIDIA GPU（8GB+ 显存）。训练 VLA 模型（Pi0、GR00T）：建议 24GB+ 显存（如 RTX 3090/4090）。轻量模型 SmolVLA 可在低配 GPU 上运行。也可以在云端训练，然后将模型下载到本地推理。

🌍 SO-101 和 SO-100 有什么区别？代码通用吗？

SO-101 是 SO-100 的改进版本，在结构件强度、伺服性能上有优化。LeRobot 代码中两者基本通用，配置文件名常以 so100 命名但同样适用于 SO-101。社区项目多数也同时标注两者兼容。
